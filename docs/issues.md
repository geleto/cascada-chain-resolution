# Issues

## Basics — do first

1. **[bug] Importing a frozen/sealed object throws.** `markShared` uses `defineProperty` (index.js:60), which throws `TypeError: object is not extensible` on frozen input — and external data is exactly where frozen objects appear. When the frozen value arrives as a promise resolution, the throw happens inside a detached continuation and becomes a host unhandled rejection (see 2). Fix: treat non-extensibility as a shared mark in itself — `hasSharedMark` gains `|| !Object.isExtensible(value)`, `markShared` returns early when not extensible. Frozen ⇒ COW-on-write, semantically exactly right.

2. **[test/code] Rejected promises assigned to properties must become Error values.** This is the important runtime behavior: data promises can reject, including promises that are already rejected when assigned, and the property/mirror must receive the resulting Error node through the normal writeback path. `settlePromise` is the load-bearing mechanism for this; every promise continuation must go through it, not raw `.then`. Exceptions thrown inside runtime continuation bodies are internal bugs and should be fatal; do not catch or convert them into language Error values. Implementation note: make fatality deterministic rather than host-policy-dependent — the single wrapper layer re-throws an escaped exception from a fresh microtask (`queueMicrotask(() => { throw err })`), so the crash is reliable regardless of the host's unhandled-rejection policy; the throw stays fatal, nothing is converted.

3. **[fixed] `importValue(promise)` breaks the one-layer wrapper contract.** `importValue(promise)` now uses `onResolve(value, v => importResolvedValue(v, rescan))`, so marking, rescanning, and rejected-promise→Error handling all go through the same promise wrapper contract as other runtime continuations.

   ```js
   function importValue(value, rescan = true) {
       if (isPromise(value)) return onResolve(value, v => importResolvedValue(v, rescan))
       return importResolvedValue(value, rescan)
   }
   ```

   (`importResolvedValue` already marks + scans, and the rejected-promise→Error path still works because `isTracked` rejects Errors.)

4. **[wontfix/spec] `assignPath(root, [], promiseValue)` may return the raw promise**, including a rejecting promise. This is valid Cascada behavior: the language error value is a special immediately-rejecting thenable, and Cascada consumers are already prepared to receive rejecting thenables/promises as error values. The sandbox represents errors with JavaScript `Error` objects in many internal paths, but the spec must not require wrapping every returned rejecting promise into an `Error` object.

5. **[fixed] Added the forked-world rejection test.** The suite now pins rejection after a COW fork: fork a pending branch, suspend writes on both forked worlds, reject the promise, and assert an Error node lands in the source and copied worlds with no unhandled rejection.

6. **[covered/no-op] Promise scheduling rule is already specified.** `initial-spec.md` already says operations run immediately, no operation is awaited before the next one is issued, and accessible promises must use `.then` and keep walking rather than pause execution. Cascada will not run this shared mutable state across workers, so no extra worker/microtask environment clause is needed.

7. **[fixed] Mirror-clearing asymmetry is commented.** Sync replacement clears the mirror; suspended replacement keeps and advances it.

## Next layer — subtree counters (replaces CLEAN; implement in this order)

8. **[decided] Subtree counters replace the CLEAN flag — lazy and branch-level.** Every *indexed* node carries exact `pendingCount`/`errorCount` totals for its subtree, so `hasError` is O(1) at a settled indexed node and `normalize` completes on the event "watched `pendingCount` hits 0" instead of rescanning. Indexing is lazy and per-branch: the first `normalize`/`hasError` on a branch indexes exactly that branch (stopping at already-indexed sub-branches, registering the boundary edge); from then on the region is maintained forever. Never-checked data pays zero bookkeeping; the first check pays the one O(branch) scan any design must pay to find the promises; repeated checks are O(1). Two load-bearing rules: indexed regions are downward-closed (indexing walks whole subtrees, writes into indexed parents index entering values — this keeps the truncated ancestor-closure cycle guard sound), and the indexed gate is evaluated at commit time, never captured at registration (continuations riding the same pending promise across indexing are safe by idempotence — the `parents` field is created only once and same-promise mirror reuse prevents duplicates). CLEAN survives only as a derived property (clean ⇔ both counts 0). Ownership/COW stays mark-based — counters answer "what is pending/broken below me", marks answer "who else can see me"; reference counting for ownership is rejected. Decisions folded in: imported Error values count as language errors; **non-extensible (frozen/sealed) nodes are forbidden to contain promises or Error values anywhere beneath them** — validated once by the boundary/indexing walk, violation commits an Error — so a frozen subtree is permanently (0,0) and carries no metadata at all (no counters, no parents, no mirrors); consequently all metadata stays inline — **one** non-enumerable Symbol property per node holding a single META record (shared flag + mirror map + counters, see 9) — no WeakMap side-table. Spec section: initial-spec.md "Subtree counters". Step-by-step implementation with pseudocode: counters-implementation.md.

9. **[code] Single META record + accessors.** Consolidate `PROMISE_MIRRORS`, `SHARED`, and the counters into one record under a single non-enumerable Symbol (`META`), never copied by `shallowCopy`. Rationale: the three concerns' divergent birth rules on COW copy get stated together in one constructor, with one hygiene rule and one hidden-class shape. (Under lazy indexing metadata is no longer dense, so this is a cleanliness win rather than a performance one — mirrors and the shared flag still share the record wherever either exists.) Shape: core fields are created up front; `parents` and `settlementWatchers` are optional fields with semantic presence:

    ```js
    function createMeta() {
        return {
            shared: false,   // set once, never cleared; false at birth = today's "no mark"
            mirrors: null,      // lazy Object.create(null) — the promise mirror map
            pendingCount: 0,
            errorCount: 0,
            settlementVerifyScheduled: false,
            // parents is added by indexBranch when counters become live:
            // undefined => not indexed; empty Map => indexed root / no indexed parents.
            // settlementWatchers is added only by normalize while callers wait.
        }
    }
    ```

    **Pitfall (pin):** record existence does not imply counted-ness — `markShared` or a mirror birth may create the record before counters are live. `parents` field presence is the marker: `parents === undefined` means unindexed, while an empty `Map` means indexed root / no indexed parents. `countsOf` tests that field, never record presence; missing `parents` triggers lazy indexing (this also covers raw roots handed to ops, e.g. the tests' `const root = {...}` pattern). Adapted accessors: `hasSharedMark(v)` = `metaOf(v)?.shared === true || !Object.isExtensible(v)`; `canUpdateMirrorToLive` reads `meta.mirrors?.[key]` (no record trivially fails the guard, same as today). Helpers: `countsOf(value)` — pending or settled-but-unreplaced promise (1,0); Error (0,1); non-extensible node (0,0) by the frozen rule; tracked node its stored totals (indexing first if `parents === undefined`); primitive (0,0). `applyCountDelta(node, dPending, dError)` — add to the node, then recurse into each parent × edge multiplicity; zero deltas short-circuit; fires settlement waiters per 12. Parent refs are strong in the sandbox (a world held by a suspended reader must keep receiving deltas) — stated honestly, this is a **time** tax, not just memory: every COW'd-away world stays referenced via reused children's edges and keeps receiving deltas, so per-delta cost grows with the COW history of a hot key. Acceptable for short-lived sandbox runs; the real implementation *requires* WeakRef edges pruned on dead deref to bound both. Propagation termination relies on acyclicity → the cycle guard in 10 is mandatory, not optional.

10. **[code] Indexing and boundary screening walkers (absorb the import rescan).** Counter indexing scans structures that become part of an indexed region: initialize counters bottom-up, register parent edges, mint a promise mirror (DISCOVERY) for every promise-valued key — eager because nothing rescans indexed regions; an orphan promise with no registered writeback would hold `pendingCount` up forever. **Transactional — two passes:** validate first (pure: two-color cycle check, frozen rule, direct back-edge target reachability — no metadata, no mirrors, no edges), commit second (metadata, counters, mirrors, edges — cannot fail). Single-pass minting would, on a late violation, leave live DAG-shared nodes carrying parent edges into the rejected value — future deltas would propagate into and retain junk. Import runs boundary screening/marking mode: every promise key gets a continuation marking its resolved value shared and re-screening it in the same mode (the ownership-boundary obligation), but import builds no counters. The `import(rescan)` parameter is **removed** — exact counters and ownership marking cannot coexist with skipping the scan (delete the "can skip promise rescan" test). Enforce the frozen rule: a non-extensible node reached by the walk must contain no promises/Errors beneath it, else commit an Error at the written key or return an Error from import. Cycle checking lives in boundary validation/indexing — cycles can enter solely through external data; internal ops cannot create them, because COW copies the target before a self-referencing set (`a.property = a` installs the pre-copy object into the new copy). Cycle guard for externally-resolved values: if the value reaches the write target, commit an Error instead — closes the writeback back-edge, the one way a cycle could enter. **Validation and indexing are separate entry points.** Eager: `screenExternalValue(target, value)` (every external writeback — frozen + cycle rules checked the moment data enters, so language invariants hold unconditionally, never "until first indexing"; value in → value-or-Error out, with the indexing commit folded in for indexed targets) and `scanImportBoundary` (import: screen + mint mirrors + mark-on-settle continuations; no counters/parents — import does not index). Lazy: `indexBranch` (the first `normalize`/`hasError` on a branch — the region creator, at the caller's program position; `setProperty` on an already-indexed parent; the commit half after writeback validation when the target is indexed). The back-edge check is ONE universal mechanism in v1: validation descends the value (through indexed nodes too when a target is present, `validated`-set-deduped) checking `node === target` before any early exit — sufficient for indexed and unindexed targets alike, because a write-created cycle must reach the written target. The ancestor closure is a stop-early optimization for indexed targets, added only if profiling demands it. `indexBranch` never throws: `normalize` surfaces a violation as its Error return and `hasError` as `true` — indexing has no parent/key to commit an Error into; only the write helpers commit Errors at keys. Traversal uses two-color marking, not a flat `seen` set: *visiting* (on the current descent stack) and *visited* (completed). Reaching a visiting node is a cycle → commit an Error; reaching a visited node is a DAG share → register the edge and reuse its totals without recursing. A flat set would let a cycle pass as a re-visit and hang delta propagation later.

11. **[code] Count bookkeeping in `setProperty`/`deleteProperty` — a separate plug-and-play module.** All counter code lives in refcounts.js, layered without circular imports: helpers.js → meta.js (new, tiny — the META record, marks) → refcounts.js → index.js. index.js only calls the hooks (`setProperty`/`deleteProperty` — void, failure handling internal; `copyCounters` at the end of `shallowCopy`; `screenExternalValue(target, value)` — value in, value-or-Error out, one line at the writeback) and implements `normalize`/`hasError` itself (~ten lines each) on top of `whenSettled` (which absorbs all frozen handling and indexing violations) and `branchHasErrors` — they are language ops needing `lookupPath`; refcounts.js's one upward need (mirror minting during indexing) is injected once at startup (`refcounts.init({ mintMirror })`). With the module stubbed to passthroughs the kernel is bit-for-bit the base kernel — unindexed behavior ≡ base-kernel behavior, testable by running the full suite against the stub. Two small helpers own exactly the counter work, nothing else — behind the lazy gate: if the written parent is not indexed (`metaOf(parent)?.parents === undefined`), they perform the bare write and skip all bookkeeping; the gate is checked when the helper *runs*, so continuations registered pre-indexing bookkeep correctly post-indexing. `setProperty(parent, key, value)`: index an entering value when needed, delta at the written node (new property → `+countsOf(new)`; replaced property → `−countsOf(old) +countsOf(new)`), swap parent edges (decrement the old tracked child's multiplicity, increment the new's), write, propagate. `deleteProperty(parent, key)`: `−countsOf(old)` — the cleared key's contribution is simply removed — remove the edge, delete, propagate; if a child's `Map<parent, edgeCount>` reaches zero for that parent, delete only that map entry, never the `parents` map itself because empty map still means indexed. Promise mirrors are deliberately *not* their concern: mirror lifecycle (BIRTH-1 fresh mirror, guarded writeback, clear on overwrite/delete) stays at the operation sites exactly as in the current kernel — the helpers need no mirror awareness because a promise physically at the key already counts (1,0) via `countsOf`, so assign/writeback/delete deltas come out right automatically. The current implementation shape (the `onTarget` callbacks, the walk) is preserved; the refactor is a greppable substitution of bare writes. Invariant: no bare `node[key] =` or `delete node[key]` outside the helpers. A generic full-recompute design (`recomputeRefCounts` on every dirtying write) was considered and rejected for the runtime path — it moves complexity (cached forward edge map, diff logic, O(width) writes) rather than removing it, since the delta rules have no case analysis to get wrong (`countsOf(undefined)` = (0,0) makes new-vs-replace one expression); it survives as the test-suite consistency oracle `verifyRefCounts` (counters-implementation.md, step 5). The caller sites: `assignPath` onTarget (a newly assigned pending promise contributes (1,0); mirror registered at assignment as today); `deletePath` onTarget; writeback under `canUpdateMirrorToLive` (replace (1,0) with `countsOf(resolvedValue)`, indexing it if needed); both walk installs (sync `next !== child` and suspended `parent[key] = next` — a revoked/disconnected commit runs the same code and propagates only through that world's ancestors, exact per world); `shallowCopy` copies language keys and forks mirrors as today, then calls `copyCounters(obj, copy)`, which is a no-op unless the source is indexed; indexed copies snapshot counts, receive `parents = new Map()`, and register themselves as parent on reused tracked children.

12. **[code] Settlement: `normalize` waits on the counters, `hasError` walks guided by them.** Two different mechanisms. Both ops index the branch on first use (`ensureIndexed` at the caller's program position, surfacing an indexing violation as a language value — `normalize` → Error, `hasError` → true, never a throw), are issued at their program position, resume through the uniform wrapper in FIFO order, and never schedule outside the wrapper (no bare `queueMicrotask`; the zero-verification rides `onResolve` on the settling promise). Both are O(1)-guided after the first indexing scan.

    **`normalize(root, segments, full=false)`** resolves the path (lookupPath rules), then **marks the reached branch shared at call time**. The mark is what pins the snapshot: later-issued ops hit it and COW away into a new world, so the watched branch can never gain promises from them — while suspended remainders of *earlier*-issued ops still land in it, because mirror advances write the promise's position in place regardless of the mark. `pendingCount === 0` is therefore the **exact** wait-set — the promises present at call time plus, recursively, promises arriving inside their resolved values. This resolves the previously open wait-set question. (No conflict with "a query must not change ownership state": normalize is not a pure query — its return escapes, so the mark was owed; applying it at the call is what makes "current state" true.) No early exit on errors: an earlier-issued remainder can still *replace* an error before settlement, so collapse-to-Error is decided only at zero. Then: `errorCount > 0` → single Error (sandbox collapse); `full` → deep copy via the old→new identity map consulted *before* recurse (visited-gate = termination, identity-reuse = DAG dedup; COW deliberately does not preserve internal aliasing, `normalize(full)` deliberately does); else return the already-marked branch. Waiter mechanics — **normalize-only** now: `meta.settlementWatchers` created only while normalize callers wait; `meta.settlementVerifyScheduled` is the coalescing latch; verification is enqueued only while waiters exist, both on a zero-crossing and on subscribe-at-zero (otherwise no delta ever comes); never fires synchronously; on run, re-check and fire only if still 0, else stay armed — a queued consumer of the same promise may have re-armed the count.

    **`hasError(root, segments)`** never marks (pure query — only a boolean escapes) and uses **no waiters**: it answers for the *current* snapshot, counter-guided. Resolve the *parent* path (broken path — intermediate Error/missing/primitive — → true; missing terminal property → false), then at the reached branch: `errorCount > 0` → **true immediately** (safe: a counted error sits under fully-settled positions, so no earlier-issued suspended remainder can still remove it — such a remainder would ride a promise on the error's path, which would make the error uncounted); `pendingCount === 0` → **false immediately** (every earlier remainder that could touch the branch rode a promise counted inside it, so the state is final w.r.t. all earlier ops); otherwise follow **only the pending promises**, found by descending only nodes with `promiseCount > 0` — settled-clean subtrees are never entered — and probe again at each settlement, at hasError's FIFO slot, *after* the writeback committed the resolved value's counts: first error wins (early exit → true), zero pending → false, newly exposed promises → followed the same way. Frozen roots/branches carry no metadata: validate the frozen subtree (pure, repeatable — frozen data is static) and treat it as settled (0,0); a WeakSet cache is a real-impl option.

    **Early-exit indexing — the partial-index invariant.** `hasError` indexes as it searches and **abandons the walk at the first error**, so it fully indexes a branch only when that branch has no error. This is safe *only* under an atomic-per-node commit rule: a node's counters **and** the `parents` edges from its children are established together, at one commit point reached after every child is processed (move `addParentEdge` after the child-accumulation loop — the current pseudocode did it inside the loop, which would leave a child pointing at a not-yet-counted parent on a mid-walk bail). Consequences: a bailed walk leaves every node either fully counted or not counted at all — never partial — and no child's edge points at an uncounted node, so **no rollback is needed** (bailing just abandons in-progress local accumulators, per your "save counts only after all children are accounted for"). Clean descendants committed before the error become disconnected counted **islands**: locally exact, and since nothing above holds an edge into an uncounted node, no delta can propagate into garbage. A later `normalize`/`hasError` from higher up re-indexes and reconnects them via the existing already-indexed-subtree boundary rule. So "subsequent calls are fast" splits: a branch probed to a *negative* answer is fully indexed → O(1) later; a branch probed to a *positive* answer is partially indexed → a re-probe re-walks only the uncounted spine to the error (still fast). Test: `verifyRefCounts` must pass after an early-exit probe (no partial node, no dangling upward edge).

## Language integration

13. **[code] Frames-as-nodes.** Treat scope frames as nodes with variables as keys, so a pending root is an ordinary promise-valued edge and the whole mirror machinery applies unchanged with path `['varName', ...path]`. Not a kernel prerequisite (the `settlePromise` re-entry at the top of each op is FIFO-correct and reuses the same resume protocol; composition works because the language rebinds the variable to each op's return), but at language level it removes the per-op derived-promise chain on pending roots.

14. **[compiler] Enforce the single-owner rule.** The kernel cannot detect a raw (unimported, unshared) promise assigned to two locations; the compiler must guarantee it never emits that — external values enter through `import`, escapes through shared-ownership `lookupPath`.
