# Issues

## Basics ŌĆö do first

1. **[bug] Importing a frozen/sealed object throws.** `markImmutable` uses `defineProperty` (index.js:60), which throws `TypeError: object is not extensible` on frozen input ŌĆö and external data is exactly where frozen objects appear. When the frozen value arrives as a promise resolution, the throw happens inside a detached continuation and becomes a host unhandled rejection (see 2). Fix: treat non-extensibility as an immutable mark in itself ŌĆö `hasImmutableMark` gains `|| !Object.isExtensible(value)`, `markImmutable` returns early when not extensible. Frozen ŌćÆ COW-on-write, semantically exactly right.

2. **[test/code] Rejected promises assigned to properties must become Error values.** This is the important runtime behavior: data promises can reject, including promises that are already rejected when assigned, and the property/mirror must receive the resulting Error node through the normal writeback path. `settlePromise` is the load-bearing mechanism for this; every promise continuation must go through it, not raw `.then`. Exceptions thrown inside runtime continuation bodies are internal bugs and should be fatal; do not catch or convert them into language Error values. Implementation note: make fatality deterministic rather than host-policy-dependent ŌĆö the single wrapper layer re-throws an escaped exception from a fresh microtask (`queueMicrotask(() => { throw err })`), so the crash is reliable regardless of the host's unhandled-rejection policy; the throw stays fatal, nothing is converted.

3. **[fixed] `importValue(promise)` breaks the one-layer wrapper contract.** `importValue(promise)` now uses `onResolve(value, v => importResolvedValue(v, rescan))`, so marking, rescanning, and rejected-promiseŌåÆError handling all go through the same promise wrapper contract as other runtime continuations.

   ```js
   function importValue(value, rescan = true) {
       if (isPromise(value)) return onResolve(value, v => importResolvedValue(v, rescan))
       return importResolvedValue(value, rescan)
   }
   ```

   (`importResolvedValue` already marks + scans, and the rejected-promiseŌåÆError path still works because `isTracked` rejects Errors.)

4. **[wontfix/spec] `assignPath(root, [], promiseValue)` may return the raw promise**, including a rejecting promise. This is valid Cascada behavior: the language error value is a special immediately-rejecting thenable, and Cascada consumers are already prepared to receive rejecting thenables/promises as error values. The sandbox represents errors with JavaScript `Error` objects in many internal paths, but the spec must not require wrapping every returned rejecting promise into an `Error` object.

5. **[fixed] Added the forked-world rejection test.** The suite now pins rejection after a COW fork: fork a pending branch, suspend writes on both forked worlds, reject the promise, and assert an Error node lands in the source and copied worlds with no unhandled rejection.

6. **[covered/no-op] Promise scheduling rule is already specified.** `initial-spec.md` already says operations run immediately, no operation is awaited before the next one is issued, and accessible promises must use `.then` and keep walking rather than pause execution. Cascada will not run this shared mutable state across workers, so no extra worker/microtask environment clause is needed.

7. **[fixed] Mirror-clearing asymmetry is commented.** Sync replacement clears the mirror; suspended replacement keeps and advances it.

## Next layer ŌĆö subtree counters (replaces CLEAN; implement in this order)

8. **[decided] Subtree counters replace the CLEAN flag.** Every tracked node carries exact `pendingCount`/`errorCount` totals for its subtree, so `hasError` is O(1) at a quiescent node and `normalize` completes on the event "watched `pendingCount` hits 0" instead of rescanning. CLEAN survives only as a derived property (clean Ōćö both counts 0). Ownership/COW stays mark-based ŌĆö counters answer "what is pending/broken below me", marks answer "who else can see me"; reference counting for ownership is rejected. Decisions folded in: imported Error values count as language errors; **non-extensible (frozen/sealed) nodes are forbidden to contain promises or Error values anywhere beneath them** ŌĆö validated once by the adoption walk, violation commits an Error ŌĆö so a frozen subtree is permanently (0,0) and carries no metadata at all (no counters, no parents, no mirrors); consequently all metadata stays inline ŌĆö **one** non-enumerable Symbol property per node holding a single META record (immutable flag + mirror map + counters, see 9) ŌĆö no WeakMap side-table. Spec section: initial-spec.md "Subtree counters". Step-by-step implementation with pseudocode: counters-implementation.md.

9. **[code] Single META record + accessors.** Consolidate `PROMISE_MIRRORS`, `IMMUTABLE`, and the counters into one record under a single non-enumerable Symbol (`META`), never copied by `shallowCopy`. Rationale: exact `parents` edges make metadata dense ŌĆö essentially every tracked node carries a record ŌĆö so the immutable flag and mirror map piggyback on the one `defineProperty` that happens anyway, and the three concerns' divergent birth rules on COW copy get stated together in one constructor. Shape (fully-shaped at creation, inner containers lazy ŌĆö no ad-hoc field addition):

    ```js
    function createMeta() {
        return {
            immutable: false,   // set once, never cleared; false at birth = today's "no mark"
            mirrors: null,      // lazy Object.create(null) ŌĆö the promise mirror map
            adopted: false,     // counters valid ONLY when true ŌĆö see pitfall below
            pendingCount: 0,
            errorCount: 0,
            parents: null,      // lazy Map<parentNode, edgeCount> (multiset ŌĆö same child under
                                // two keys of one parent has multiplicity 2)
            watchers: null,     // lazy; only on normalize/hasError targets (see 12)
        }
    }
    ```

    **Pitfall (pin):** record existence does not imply counted-ness ŌĆö `markImmutable` or a mirror birth may create the record before adoption, so `countsOf` must test `adopted`, never presence; `adopted: false` triggers the adoption walk lazily (this also covers raw roots handed to ops without prior adoption, e.g. the tests' `const root = {...}` pattern). Adapted accessors: `hasImmutableMark(v)` = `metaOf(v)?.immutable === true || !Object.isExtensible(v)`; `canUpdateMirrorToLive` reads `meta.mirrors?.[key]` (no record trivially fails the guard, same as today). Helpers: `countsOf(value)` ŌĆö pending or settled-but-unreplaced promise (1,0); Error (0,1); non-extensible node (0,0) by the frozen rule; tracked node its stored totals (adopting first if `!adopted`); primitive (0,0). `applyCountDelta(node, dPending, dError)` ŌĆö add to the node, then recurse into each parent ├Ś edge multiplicity; zero deltas short-circuit; fires zero-watchers per 12. Parent refs are strong in the sandbox (a world held by a suspended reader must keep receiving deltas) ŌĆö stated honestly, this is a **time** tax, not just memory: every COW'd-away world stays referenced via reused children's edges and keeps receiving deltas, so per-delta cost grows with the COW history of a hot key. Acceptable for short-lived sandbox runs; the real implementation *requires* WeakRef edges pruned on dead deref to bound both. Propagation termination relies on acyclicity ŌåÆ the cycle guard in 10 is mandatory, not optional.

10. **[code] Adoption walker (absorbs the import rescan).** One eager scan for any structure entering tracking: initialize counters bottom-up, register parent edges, mint a promise mirror (DISCOVERY) for every promise-valued key ŌĆö eager because nothing rescans anymore; an orphan promise with no registered writeback would hold `pendingCount` up forever. Enforce the frozen rule: a non-extensible node reached by the walk must contain no promises/Errors beneath it, else commit an Error at the adopting key. Adoption is the **only** place cycle checking exists ŌĆö cycles can enter solely through external data; internal ops cannot create them, because COW copies the target before a self-referencing set (`a.property = a` installs the pre-copy object into the new copy). Cycle guard for externally-resolved values: if the adopted value contains the target or one of its ancestors, commit an Error instead ŌĆö closes the writeback back-edge, the one way a cycle could enter. The ancestor closure (from `parents`, exact now) is computed *lazily*, only when the walk actually encounters an already-adopted node ŌĆö ancestors are always adopted, and fresh external data contains none, so the hot writeback path never pays for it. Callers: `import`, `assignPath` receiving a raw untracked structure (one-time O(size)), writeback of a resolved value containing nested structure. Traversal uses two-color marking, not a flat `seen` set: *visiting* (on the current descent stack) and *visited* (completed). Reaching a visiting node is a cycle ŌåÆ commit an Error; reaching a visited node is a DAG share ŌåÆ register the edge and reuse its totals without recursing. The two guards are complementary: coloring catches cycles inside the adopted value; the ancestor-closure check catches back-edges into the existing tree. A flat set would let a cycle pass as a re-visit and hang delta propagation later.

11. **[code] Count bookkeeping in `setProperty`/`deleteProperty`.** Two small helpers own exactly the counter work, nothing else. `setProperty(parent, key, value)`: adopt an entering value, delta at the written node (new property ŌåÆ `+countsOf(new)`; replaced property ŌåÆ `ŌłÆcountsOf(old) +countsOf(new)`), swap parent edges (decrement the old tracked child's multiplicity, increment the new's), write, propagate. `deleteProperty(parent, key)`: `ŌłÆcountsOf(old)` ŌĆö the cleared key's contribution is simply removed ŌĆö remove the edge, delete, propagate. Promise mirrors are deliberately *not* their concern: mirror lifecycle (BIRTH-1 fresh mirror, guarded writeback, clear on overwrite/delete) stays at the operation sites exactly as in the current kernel ŌĆö the helpers need no mirror awareness because a promise physically at the key already counts (1,0) via `countsOf`, so assign/writeback/delete deltas come out right automatically. The current implementation shape (the `onTarget` callbacks, the walk) is preserved; the refactor is a greppable substitution of bare writes. Invariant: no bare `node[key] =` or `delete node[key]` outside the helpers. A generic full-recompute design (`recomputeRefCounts` on every dirtying write) was considered and rejected for the runtime path ŌĆö it moves complexity (cached forward edge map, diff logic, O(width) writes) rather than removing it, since the delta rules have no case analysis to get wrong (`countsOf(undefined)` = (0,0) makes new-vs-replace one expression); it survives as the test-suite consistency oracle `verifyRefCounts` (counters-implementation.md, step 5). The caller sites: `assignPath` onTarget (a newly assigned pending promise contributes (1,0); mirror registered at assignment as today); `deletePath` onTarget; writeback under `canUpdateMirrorToLive` (replace (1,0) with `countsOf(resolvedValue)`, adopting it); both walk installs (sync `next !== child` and suspended `parent[key] = next` ŌĆö a revoked/disconnected commit runs the same code and propagates only through that world's ancestors, exact per world); `shallowCopy` (builds the copy's entire META record in one `createMeta`-based expression, the divergent birth rules co-located: `immutable: false` ŌĆö the copy is owned; fresh mirror map with eagerly forked mirrors; counter totals snapshotted from the source ŌĆö identical keys ŌćÆ identical totals, including (1,0) per forked promise key, and the source must be adopted first; empty `parents`, its own edge added when the caller installs it; no watchers. The copy registers itself as a parent on every reused tracked child at copy time).

12. **[code] Zero-watchers, `normalize`, `hasError`.** `whenQuiescent(node)`: watchers live on the counter record; a verification microtask is enqueued both when a commit drops a watched `pendingCount` to 0 *and* when a subscription arrives while `pendingCount` is already 0 (otherwise no delta ever comes and the watcher never fires); never fire synchronously ŌĆö uniformly async, and on run re-check and fire only if still 0, else stay armed ŌĆö required because the zero-crossing happens inside one continuation of a promise whose already-queued later consumers (suspended remainders of earlier-issued ops) may install new promises. This is the O(1)-per-iteration replacement of the spec's rescan loop. `normalize(root, full=false)`: settle a promise/primitive root per existing op semantics; else await quiescence, then `errorCount > 0` ŌåÆ single Error (sandbox collapse); `full` ŌåÆ deep copy via an oldŌåÆnew identity map consulted *before* recurse (one map, two stated rules: visited-gate = termination, identity-reuse = DAG dedup ŌĆö lookup-shared diamonds must not duplicate; COW deliberately does not preserve internal aliasing, `normalize(full)` deliberately does); else mark the root immutable and return it. `hasError(root, segments)`: resolve the path with the lookupPath machinery (true on path failure), then await the reached branch's quiescence, then `errorCount > 0` ŌĆö waiting is required for program-order semantics: a pending remainder of an earlier op may still add or remove an error, and any such remainder is suspended on a promise counted inside the branch.

## Language integration

13. **[code] Frames-as-nodes.** Treat scope frames as nodes with variables as keys, so a pending root is an ordinary promise-valued edge and the whole mirror machinery applies unchanged with path `['varName', ...path]`. Not a kernel prerequisite (the `settlePromise` re-entry at the top of each op is FIFO-correct and reuses the same resume protocol; composition works because the language rebinds the variable to each op's return), but at language level it removes the per-op derived-promise chain on pending roots.

14. **[compiler] Enforce the single-owner rule.** The kernel cannot detect a raw (unimported, unshared) promise assigned to two locations; the compiler must guarantee it never emits that ŌĆö external values enter through `import`, escapes through shared-ownership `lookupPath`.
