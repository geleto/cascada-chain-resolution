# Issues

## Basics — do first

1. **[bug] Importing a frozen/sealed object throws.** `markImmutable` uses `defineProperty` (index.js:60), which throws `TypeError: object is not extensible` on frozen input — and external data is exactly where frozen objects appear. When the frozen value arrives as a promise resolution, the throw happens inside a detached continuation and becomes a host unhandled rejection (see 2). Fix: treat non-extensibility as an immutable mark in itself — `hasImmutableMark` gains `|| !Object.isExtensible(value)`, `markImmutable` returns early when not extensible. Frozen ⇒ COW-on-write, semantically exactly right.

2. **[test/code] Rejected promises assigned to properties must become Error values.** This is the important runtime behavior: data promises can reject, including promises that are already rejected when assigned, and the property/mirror must receive the resulting Error node through the normal writeback path. `settlePromise` is the load-bearing mechanism for this; every promise continuation must go through it, not raw `.then`. Exceptions thrown inside runtime continuation bodies are internal bugs and should be fatal; do not catch or convert them into language Error values.

3. **[fixed] `importValue(promise)` breaks the one-layer wrapper contract.** `importValue(promise)` now uses `onResolve(value, v => importResolvedValue(v, rescan))`, so marking, rescanning, and rejected-promise→Error handling all go through the same promise wrapper contract as other runtime continuations.

   ```js
   function importValue(value, rescan = true) {
       if (isPromise(value)) return onResolve(value, v => importResolvedValue(v, rescan))
       return importResolvedValue(value, rescan)
   }
   ```

   (`importResolvedValue` already marks + scans, and the rejected-promise→Error path still works because `isTracked` rejects Errors.)

4. **[wontfix/spec] `assignPath(root, [], promiseValue)` may return the raw promise**, including a rejecting promise. This is valid Cascada behavior: the language error value is a special immediately-rejecting thenable, and Cascada consumers are already prepared to receive rejecting thenables/promises as error values. The sandbox represents errors with JavaScript `Error` objects in many internal paths, but the spec must not require wrapping every returned rejecting promise into an `Error` object.

5. **[fixed] Added the forked-world rejection test.** The suite now pins rejection after a COW fork: fork a pending branch, suspend writes on both forked worlds, reject the promise, and assert an Error node lands in the source and copied worlds with no unhandled rejection.

6. **[covered/no-op] Promise scheduling rule is already specified.** `initial-spec.md` already says operations run immediately, no operation is awaited before the next one is issued, and accessible promises must use `.then` and keep walking rather than pause execution. Cascada will not run this shared mutable state across workers, so no extra worker/microtask environment clause is needed.

7. **[fixed] Mirror-clearing asymmetry is commented.** Sync replacement clears the mirror; suspended replacement keeps and advances it.

## Next layer — subtree counters (replaces CLEAN; implement in this order)

8. **[decided] Subtree counters replace the CLEAN flag.** Every tracked node carries exact `pendingCount`/`errorCount` totals for its subtree, so `hasError` is O(1) at a quiescent node and `normalize` completes on the event "watched `pendingCount` hits 0" instead of rescanning. CLEAN survives only as a derived property (clean ⇔ both counts 0). Ownership/COW stays mark-based — counters answer "what is pending/broken below me", marks answer "who else can see me"; reference counting for ownership is rejected. Decisions folded in: imported Error values count as language errors; **non-extensible (frozen/sealed) nodes are forbidden to contain promises or Error values anywhere beneath them** — validated once by the adoption walk, violation commits an Error — so a frozen subtree is permanently (0,0) and carries no metadata at all (no counters, no parents, no mirrors); consequently all metadata stays in inline non-enumerable Symbol properties like `PROMISE_MIRRORS` — no WeakMap side-table. Spec section: initial-spec.md "Subtree counters".

9. **[code] Counter metadata + accessors.** Per tracked extensible node, under one non-enumerable Symbol (same hygiene as `PROMISE_MIRRORS`, never copied by `shallowCopy`): `pendingCount`, `errorCount`, `parents: Map<parentNode, edgeCount>` (multiset — the same child under two keys of one parent has multiplicity 2), `watchers` (see 12). Helpers: `countsOf(value)` — pending or settled-but-unreplaced promise (1,0); Error (0,1); non-extensible node (0,0) by the frozen rule; tracked node its stored totals (adopting first if unknown); primitive (0,0). `applyCountDelta(node, dPending, dError)` — add to the node, then recurse into each parent × edge multiplicity; zero deltas short-circuit; fires zero-watchers per 12. Parent refs are strong in the sandbox (a world held by a suspended reader must keep receiving deltas); WeakRef edges are a real-implementation memory optimization. Propagation termination relies on acyclicity → the cycle guard in 10 is mandatory, not optional.

10. **[code] Adoption walker (absorbs the import rescan).** One eager scan for any structure entering tracking: initialize counters bottom-up, register parent edges, mint a promise mirror (DISCOVERY) for every promise-valued key — eager because nothing rescans anymore; an orphan promise with no registered writeback would hold `pendingCount` up forever. Enforce the frozen rule: a non-extensible node reached by the walk must contain no promises/Errors beneath it, else commit an Error at the adopting key. Cycle guard for externally-resolved values: compute the target's ancestor closure from `parents` (exact now); if the adopted value contains the target or one of its ancestors, commit an Error instead — closes the writeback back-edge, the one way a cycle could enter. Callers: `import`, `assignPath` receiving a raw untracked structure (one-time O(size)), writeback of a resolved value containing nested structure. Guarded by a `seen` set.

11. **[code] Deltas at every commit point.** Every runtime property write applies `delta = countsOf(new) − countsOf(old)` at the written node, swaps parent edges (decrement the old tracked child's multiplicity, increment the new's), and propagates. The sites: `assignPath` onTarget (a newly assigned pending promise contributes (1,0); mirror registered at assignment as today); `deletePath` onTarget; writeback under `canUpdateMirrorToLive` (replace (1,0) with `countsOf(resolvedValue)`, adopting it); both walk installs (sync `next !== child` and suspended `parent[key] = next` — a revoked/disconnected commit runs the same code and propagates only through that world's ancestors, exact per world); `shallowCopy` (copy's counters start as a snapshot of the source's totals — identical keys ⇒ identical totals, including (1,0) per forked promise key; copy is born with empty `parents`, its own edge added when the caller installs it; copy registers itself as a parent on every reused tracked child at copy time).

12. **[code] Zero-watchers, `normalize`, `hasError`.** `whenQuiescent(node)`: watchers live on the counter record; when a commit drops a watched `pendingCount` to 0, enqueue a verification microtask; on run re-check and fire only if still 0, else stay armed — required because the zero-crossing happens inside one continuation of a promise whose already-queued later consumers (suspended remainders of earlier-issued ops) may install new promises. This is the O(1)-per-iteration replacement of the spec's rescan loop. `normalize(root, full=false)`: settle a promise/primitive root per existing op semantics; else await quiescence, then `errorCount > 0` → single Error (sandbox collapse); `full` → deep copy via an old→new identity map consulted *before* recurse (one map, two stated rules: visited-gate = termination, identity-reuse = DAG dedup — lookup-shared diamonds must not duplicate; COW deliberately does not preserve internal aliasing, `normalize(full)` deliberately does); else mark the root immutable and return it. `hasError(root, segments)`: resolve the path with the lookupPath machinery (true on path failure), then await the reached branch's quiescence, then `errorCount > 0` — waiting is required for program-order semantics: a pending remainder of an earlier op may still add or remove an error, and any such remainder is suspended on a promise counted inside the branch.

## Language integration

13. **[code] Frames-as-nodes.** Treat scope frames as nodes with variables as keys, so a pending root is an ordinary promise-valued edge and the whole mirror machinery applies unchanged with path `['varName', ...path]`. Not a kernel prerequisite (the `settlePromise` re-entry at the top of each op is FIFO-correct and reuses the same resume protocol; composition works because the language rebinds the variable to each op's return), but at language level it removes the per-op derived-promise chain on pending roots.

14. **[compiler] Enforce the single-owner rule.** The kernel cannot detect a raw (unimported, unshared) promise assigned to two locations; the compiler must guarantee it never emits that — external values enter through `import`, escapes through shared-ownership `lookupPath`.
