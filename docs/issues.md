# Issues

## Subtree Counters

1. **Implemented: subtree counters replace the CLEAN flag.** `src/refcounts.js` owns exact `promiseCount`/`errorCount` totals for ref-indexed branches. Ref-indexing is lazy and per branch; never-checked data pays zero counter bookkeeping. Writes and deletes use prepared edge transitions, draining promise mirrors replace `[1,0]` with their final state once, and indexed COW copies reconstruct counters from their own logical placements.

    Implemented rules:

    - Ref-indexed regions are downward-closed: ref-indexing walks whole subtrees, and writes into ref-indexed parents ref-index the entering value before commit.
    - The ref-indexed gate is evaluated at commit time, never captured at registration time.
    - `parents === undefined` means not ref-indexed; an empty `Map` means ref-indexed root / no ref-indexed parents.
    - CLEAN is only derived: clean means `promiseCount === 0 && errorCount === 0`.
    - Ownership/COW stays mark-based; counters answer "what is pending/broken below me", and the SHARED mark answers "who else can see me".
    - Error values count as language errors inside ref-indexed branches.
    - Non-extensible nodes follow the same counter rules as extensible nodes. Promises and Errors inside them are valid, and metadata uses the WeakMap fallback.
    - Counter metadata, promise mirrors, and the SHARED mark share the single `META` record; `src/meta.js` owns generic metadata and `src/promise-mirrors.js` owns mirror lifecycle.
    - `normalize` is implemented by issue 6 and `hasError` by issue 7; they activate this ref-indexing at their public operation boundary.

2. **Implemented: single META record and accessors.** `src/meta.js` owns the one logical metadata record. Storage is selected by `STORE_META_IN_WEAKMAP`, derived from `CASCADA_META_STORAGE`: inline non-enumerable Symbol property by default, or WeakMap when set to `weakmap`. Shared marks, promise mirrors, and subtree counters all use that record, and `shallowCopy` never copies it as language data. The default test command runs the complete suite once in each mode.

    ```js
    function createMeta() {
        return {
            shared: false,   // set once, never cleared; false at birth = "no mark"
            mirrors: null,   // lazy Object.create(null), the promise mirror map
            promiseCount: 0,
            errorCount: 0,
            settlementPromise: undefined,
            settlementResolve: undefined,
            importContext: undefined,
            importPrepared: false,
            cycleErrors: null, // lazy owner/key map of cycle Error cuts
            // parents is added by buildRefIndex when counters become live:
            // undefined => not ref-indexed; empty Map => ref-indexed root / no ref-indexed parents.
        }
    }
    ```

    Implemented rules:

    - `metaOf(value)` returns only the node's own metadata. Inline mode uses an own Symbol on extensible nodes and the same WeakMap fallback as WeakMap mode for non-extensible nodes.
    - `hasSharedMark(value)` reads `metaOf(value)?.shared === true || !Object.isExtensible(value)`.
    - Promise mirrors live in `meta.mirrors`; `src/promise-mirrors.js` creates the map lazily as `Object.create(null)` and owns mirror lifecycle.
    - Counter fields live directly on META. `parents === undefined` remains the ref-indexing gate; record existence alone does not mean counters are live.
    - COW copies receive no copied META object. If counters are live, `copyCounters` creates fresh metadata and reconstructs counts from the copy's logical placements.

3. **Superseded by items 11–12: eager import screening.** Import is root-only; imported alias discovery and cycle analysis are lazy and live in `src/import.js`.

4. **Superseded by items 11–12: recursive import marking.** Imported descendants are discovered only at first counter demand, and non-extensible metadata uses the common WeakMap fallback in both storage modes. Full design: `docs/lazy-import.md`.

5. **Superseded by item 12: separate write bookkeeping helpers.** Property replacement uses `prepareEdgeTransition` before commit, while assignment, deletion, mark publication, and mirror drain all share one atomic live-edge bookkeeping primitive. Full design: `docs/counters-implementation.md`.

6. **Implemented: `normalize` and settlement.** normalize is lookup-shaped: it resolves the path with lookup rules through the observational, context-threading `walkObservationPath`, then marks exactly what escapes plus any pending branch that must be pinned while it settles. It ref-indexes the branch on first use at the caller's program position. It follows the kernel's value-or-promise convention: whenever the answer is decided before any suspension, it is returned synchronously. All internal waits go through the uniform wrapper in FIFO order — settlement waits are consumed through `onInternalResolve`, never a bare `.then` outside helpers. (These conventions are shared with `hasError`, issue 7.) Placement: the settlement-wait machinery lives in `src/refcounts.js` next to `applyCountDelta`, which fires its only trigger; `normalize` and `copyToPlainValue` are ops in `src/index.js`. `copyToPlainValue` is the third sanctioned write bypass (plain output data, no metadata to bookkeep) and carries the pinned comment like the other two.

    `normalize(chain, segments, sharedOwnership=true, plainCopy=false)` — resolve, ref-index, answer if settled, otherwise pin and wait:

    - Resolve the path with `walkObservationPath(chain, segments, onResolved)` (reused by hasError, issue 7): callers decide marking after resolution. The resolver threads importContext, both to flavor discovered mirrors and to attribute failures. An existing Error is propagated; a broken required prefix becomes a path-access Error; an untracked final target (primitive/null/undefined/missing) is returned as-is — no counting, no marks, no settlement wait. A promise along the way is continued through its mirror.
    - Ref-index the reached branch, passing the walk's inherited import context into `buildRefIndex` so imported aliases and cycles are prepared with the correct attribution.
    - **Mark exactly what escapes, plus mark to pin a wait** — `markImported` with context inside an imported region; otherwise `markShared` only when `sharedOwnership=true`, except pending normalize always marks to pin the wait. A settled `errorCount > 0` returns a single Error **unmarked** (only an Error escapes); a settled `plainCopy=true` returns the deep copy **unmarked** (the copy is independent and produced inside the synchronous prefix, so nothing can interleave with it; copies are born owned).
    - If `promiseCount === 0` already, the answer is synchronous: `errorCount > 0` → a single Error; `plainCopy=true` → the deep copy; otherwise mark according to `sharedOwnership` and return the branch. No settlement wait is ever created for a settled branch, so the zero-count wait case does not exist.
    - Otherwise mark regardless of `sharedOwnership` (the pin) and wait. The pin makes the wait-set exact: later-issued operations COW away from the marked branch, while suspended remainders of earlier operations still land through mirror advances. Under issue 12 each unresolved or consumer-draining mirror edge remains `[1,0]`, so `promiseCount === 0` means both the recursively exposed promises and every registered advance that can affect the snapshot have drained. Do not collapse to Error until settlement, because an earlier-issued remainder may still replace an Error before zero; a branch that later collapses to Error keeps its pin mark, since the outcome is unknowable at call time.
    - `meta.settlementPromise` is one shared settlement promise generation and exists only while normalize callers are waiting; `meta.settlementResolve` holds its resolver; the zero-crossing in `applyCountDelta` clears and resolves that generation. Liveness holds because every decrement of the pinned branch's counters happens inside a continuation of one of its counted promises: later-issued operations COW away, and earlier-issued remainders ride counted promises — their writebacks and their COW installs at mirrored keys (which may remove counted promises without settling them) all run as continuations of promises the branch counts.
    - Snapshot exactness also leans on the compiler disciplines of issue 15 (import routing and single-owner lowering): a raw promise illegally shared between a location inside the pinned branch and one outside it could mutate the branch outside its counted wait-set.
    - Issue 12 makes the zero-crossing exact: a mirror stays logically `[1,0]` until its registered consumers drain, so zero cannot occur while an earlier advance is merely queued. The settlement generation is therefore resolved directly at that transition; its Promise continuations remain asynchronous.
    - On fire: `errorCount > 0` returns a single Error; `plainCopy=true` returns a deep copy through an old->new identity map consulted before recursing (preserving own enumerable `__proto__` data keys through safe own-property writes); otherwise return the already-marked branch.

    A reached non-extensible root is ref-indexed like any other tracked value and creates no settlement wait only when its `promiseCount` is already zero. Non-extensible nodes store counters, mirrors, and placement metadata in the WeakMap fallback.

    `plainCopy=false` is a language-internal normalization result: it exposes the branch's logical settled state to runtime operations and preserves its identity, ownership marks, and metadata. An external imported or non-extensible holder may still physically contain its original, already-settled Promise while its mirror supplies the logical value. Host-facing output, serialization, and renderers must therefore request `plainCopy=true` (or remain entirely on lookup-driven runtime access); only the plain copy guarantees ordinary physical data with logical settled values materialized and no runtime metadata.

    Issue 12 adds one imported-cycle exception to the implemented `errorCount > 0` collapse: a cached cycle Error still contributes `[0,1]` for hasError/getErrors and counter termination, but normalize follows the raw cyclic value with a cycle-aware identity/promise walk. Genuine Errors retain the behavior above.

7. **Implemented: `hasError`.** Implemented after normalize (issue 6), which it builds on: the counters, the settlement conventions, the non-marking context-threading `walkObservationPath`, and the ref-indexing walk already exist. `hasErrorAtPathValue` builds the generic ref index only for the value reached at the path, then `hasErrorInIndexedBranch` answers settled branches directly from `errorCount` and `promiseCount`. Only a pending branch allocates the final boolean race and collects promise waits. It is a pure query with no shared settlement wait and no marks, and returns a bare boolean whenever the answer is decided before any suspension.

    Empty path probes the root itself; non-empty paths use `walkObservationPath(chain, path, onResolved)`. A missing final target reaches the callback as `undefined` and is false. Missing/null/undefined or primitive intermediates become a language Error inside the resolver and are true; an existing Error is propagated. All property reads, promise handling, and import-context threading therefore remain inside `walkObservationPath`. No extraction marking occurs: only a boolean escapes (issue 4's lookupPath would mark; hasError must not, and unlike normalize it never marks the reached branch either). `__proto__` and own non-enumerable segments follow lookup semantics — missing as a target, Error when another segment tries to dereference them.

    Deliberately no pin: marks are irreversible, so pinning for a boolean would permanently convert every queried branch to COW-on-write — a query must not change the program's write behavior. And the wait tree makes a pin unnecessary for exactness: the wait set is captured at issue time and extended only through resolved values read at hasError's own FIFO slots, so the answer reflects the issue-time branch plus its recursive exposures. Later-issued installs are simply invisible (the "original indexed frontier" test pins this); later-issued overwrite/delete can detach a watched promise's resolved value from the live tree, but the captured mirror still answers for the issue-time branch. This is the query contract. At the reached branch:

    - `hasErrorAtPathValue` calls generic `buildRefIndex` at the path-value boundary only. The branch is fully indexed, preserving the invariant that a ref-indexed parent never has a tracked child without counters.
    - On the indexed branch, `errorCount > 0` -> true immediately (a counted error sits under settled positions; no earlier-issued remainder can remove it). `promiseCount === 0` -> false immediately (every earlier remainder rode a counted promise). Both are bare booleans.
    - Pending handling starts in `collectErrorSearchWaits`: after `buildRefIndex` succeeds and only when `promiseCount > 0`, it descends the ref-indexed branch through nodes whose counters still contain promise/mirror work and pushes one wait per pending key through the required existing mirror: `onPromiseMirrorResolve(mirror, () => probeResolvedPromiseForErrors(mirror.currentValue, resolveError))`. Registering waits after `buildRefIndex` is load-bearing: preparation minted and registered mirror writeback first, so hasError consumes the value after every earlier FIFO advance. Behind a promise barrier, `probeIndexedBranchForErrors` checks the prepared branch's counters and returns any nested readiness tree. The shared collector builds each `Promise.all(waitPromises)`, and hasError returns `Promise.race([errorPromise, onInternalResolve(cleanPromise, () => false)])`.
    - Promise-mirror preparation guarantees that, before each later FIFO consumer runs, `currentValue` is an untracked value/Error or a ref-indexed tracked branch, with any cycle Error held in the mirror's placement metadata. Issue 12 keeps this preparation private while consumers drain: a live mirror attaches only its final state, while a revoked mirror never touches the former parent property or counts. Each hasError settlement continuation checks the prepared placement counts, handles clean scalar cases directly, and probes an indexed tracked `currentValue`; it never re-enters `hasErrorAtPathValue`. If it sees more issue-time promises, it returns that nested `Promise.all`. When the whole promise tree drains, the clean side resolves false.
    - One call-local `visited` WeakSet is shared through the complete promise tree. Only nodes with their own shared status are checked and added. It does not retain settled branches while another captured promise remains pending. Imported preparation marks every retained imported node shared, so repeated external identities can be skipped safely. A trusted identity cannot recur inside one branch under the compiler's single-owner/COW contract.
    - Accepted corner: a later-issued overwrite/delete of a pending key decrements live counts synchronously but wakes nothing until the revoked promise settles — the answer is delayed, not wrong (that promise was in the wait-set as of hasError's issue position, exactly what sequential execution waits for); a never-settling revoked promise leaves hasError pending.

    A reached non-extensible branch uses ordinary counters; contained Promises are awaited and contained Errors return `true`.

8. **Implemented: fatal error reporting.** Internal/runtime failures go through the global fatal boundary: `reportFatalError(error)` reports and throws. Fatal errors are never language Error values and must never be confused with rejected data promises, which continue to be converted by `onValueResolve` into language Error values at promise-settlement boundaries.

    Required shape:

    - `src/error.js` exports `reportFatalError(error)` and `setFatalErrorReporter(reporter)`; the sandbox reporter is a no-op, and `reportFatalError` always throws the original error after reporting.
    - An object-like fatal value is reported once per identity even when it unwinds through nested wrapper layers; every `reportFatalError` call still throws the original value.
    - Internal invariant failures, compiler-contract violations, continuation throws, and late failures in hasError/normalize wait branches call `reportFatalError`.
    - `onValueResolve` still has exactly one data-facing catch: promise rejection becomes the language Error value before the continuation runs. Exceptions thrown by the continuation, failures while converting a rejection reason, and promises rejected by the continuation are fatal and routed through `reportFatalError`.
    - Issue 12 collapses `onValueResolve` to one reaction on the source promise: the fulfilled branch calls `runFatal(fn, value)`, while the rejected branch performs `errorFromRejection` and invokes `fn` inside the same `runFatal` call so conversion failures remain fatal. Do not resolve an intermediate proxy and invoke `fn` in a second reaction. One-stage value and internal consumers then preserve their registration order when mixed on the same promise; mirror-specific draining supplies the additional guard needed by synchronous settled reads.
    - `onInternalResolve` is for runtime aggregate promises such as hasError's clean wait tree: source rejection or a promise rejected by its continuation is fatal, not converted to a language Error. Races that may answer before all internal waits finish attach this fatal handler to the losing waits, so a later kernel bug is reported even after the public operation result has already settled.
    - A fatal in a discarded internal mutator continuation is reported and leaves its derived promise rejected. Mutators deliberately return nothing, so that rejection reaches the host as unhandled; this is the fatal contract, not a language Error conversion.

9. **Implemented: Chain root state location.** Cascada roots are mutable locations, not bare values. A pending root promise therefore still needs a stable parent/key where normal promise mirrors can live. The sandbox now exports `Chain`; public operators receive a `Chain`, immediately operate on the private `chain._state.value` holder slot, and never walk or ref-index the `Chain` object itself.

    Implemented shape:

    - Public operators receive the `Chain` and operate on `chain._state.value` as the root value.
    - Mutating operators return nothing. They synchronously replace `chain._state.value` when the root is available; when it is a pending promise, the ordinary promise mirror for `_state.value` serializes root-level operations and commits its final drained value only while that mirror remains live. Values are observed only through lookupPath, normalize, hasError, or getErrors; the Chain slot remains the authoritative root location.
    - Bookkeeping functions (`buildRefIndex`, `markShared`, `markImported`, promise mirrors, COW, normalize/hasError branch probes) receive only values below `_state.value`, never `chain`; the private `_state` holder can carry normal META, but it is host state, not language data.
    - Other `Chain` fields such as command arrays, caches, schedulers, and bookkeeping are outside the language object graph. They are not counted, copied, marked, mirrored, or validated by this kernel.

    This keeps the root promise ordering problem solved without turning host runtime objects into Cascada data. A call shaped like `assignPath(chain._state.value, path, value)` is invalid for mutable roots because it cannot update the root location; the valid call shape is `assignPath(chain, path, value)`.

10. **Implemented: `getErrors` collects all errors in a branch.** `getErrors(chain, path)` returns the distinct Error values reachable in the issue-time branch at `path`. It resolves the full path through observational `walkObservationPath`: an existing Error encountered along the path is propagated, while a missing/null/undefined or primitive intermediate becomes a new path-access Error. An Error-blocked or broken path therefore returns that Error. Any successfully reached terminal with no reachable Errors returns `[]`, including missing, null/undefined, primitive, and clean tracked terminals regardless of extensibility. No path or position metadata is returned.

    The only operation-level `buildRefIndex` call is at the reached path-value boundary, with the inherited import context. Every tracked node entering the collector is already ref-indexed. The operation is a pure query: no shared mark and no settlement pin. Its promise frontier is captured at its program position and extended only through values observed in getErrors' FIFO slots, so later-issued installs are invisible and a later overwrite/delete does not discard a promise already captured by the query.

    Promise-mirror preparation preserves the indexing invariant for both live and revoked mirrors. Before a later FIFO consumer sees `mirror.currentValue`, it is an untracked value/Error or a ref-indexed tracked branch, with cycle Errors represented by mirror placement metadata. A live mirror attaches only the final drained state; a revoked mirror performs the same private preparation/ref-indexing without writing the holder or attaching the private branch to it. This responsibility stays at the promise-mirror/refcount boundary rather than making each observer call `buildRefIndex`.

    `hasError` already consumes this guarantee: `hasErrorAtPathValue` is used only for the value delivered by `walkObservationPath`, where the one operation-level `buildRefIndex` belongs. `probeResolvedPromiseForErrors` consumes the already-settled `mirror.currentValue` directly: Error signals true, primitives are clean, and every tracked value must already carry counters before `probeIndexedBranchForErrors` probes it. It never routes resolved promise contents back through `hasErrorAtPathValue`.

    hasError and getErrors share `collectErrorSearchWaits`. It always follows branches with `promiseCount > 0`; when `errors` is supplied, it also follows branches with `errorCount > 0` and adds Error-valued keys to that Set. A tracked node with no relevant counts is skipped, and every tracked child reached through an ordinary indexed edge is required to have a counter.

    For each promise key in the captured frontier, the helper requires its existing mirror, registers one `onPromiseMirrorResolve(mirror, () => onPromiseValue(mirror.currentValue))` continuation, and collects the derived wait. `onPromiseValue` performs the operation-specific work after that FIFO slot and may return another readiness promise, so `Promise.all` recursively represents the complete promise tree. The helper returns `undefined` when it collected no waits, otherwise `Promise.all(waitPromises)`. hasError omits `errors`, because it checks `errorCount > 0` in O(1) before calling the helper, and its callback retains the first-error resolver and race. getErrors passes its shared Error Set, and its callback collects from each already-indexed resolved value without an early answer or race. The outer getErrors operation returns `[...errors]` immediately for `undefined`, or only after the readiness promise resolves; even if synchronous errors are found, every captured promise must settle before the complete array is known.

    Accumulate Errors in one call-local `Set`, and share the marked-node `visited` WeakSet through every recursive promise slot. Only nodes with their own shared status enter `visited`; issue 11's lazy imported preparation gives every retained external descendant such a mark, so imported DAGs are traversed once per identity rather than once per path. Weak visitation does not retain settled branches while another captured promise remains pending. Error identity is the result deduplication rule: the same Error reachable at multiple keys appears once, while distinct Error instances remain distinct. Result order and structural position are not semantic; tests compare membership rather than settlement-dependent insertion order. At the same program position and for the same exact path's projected terminal placement, `hasError(chain, path) === (getErrors(chain, path).length > 0)`, including missing and primitive paths. This does not imply that an ancestor query collects raw Errors reachable only behind a marked counter cut; a longer path may deliberately cross that cut and establishes a different terminal placement.

    The two operation shells remain separate in `src/index.js`: they share `walkObservationPath`, counters, mirror access, fatal indexed-branch assertions, and `collectErrorSearchWaits`, without parameterizing hasError's first-error race or getErrors' exhaustive result policy. Coverage includes synchronous nested errors; errors behind multiple promise barriers; rejection conversion; same Error at multiple keys and through a DAG; missing, primitive, Error-blocked, frozen, and cyclic paths; a known Error plus an unresolved promise remaining pending; later-issued installs remaining invisible; overwritten/deleted captured mirrors still contributing their private results; concurrent queries with independent issue-time frontiers; coexistence with a normalize settlement pin; no shared mark or settlement pin under either metadata storage mode; and `hasError(chain, path) === (getErrors(chain, path).length > 0)` at the same logical program position, synchronously and behind settling promise barriers, including broken required prefixes.

11. **Implemented with item 12: lazy imported-subtree preparation and imported DAG deduplication.** Import now marks only its boundary root, and descendant marks needed by error-query deduplication are materialized only when an imported region is first ref-indexed. Variables that never use normalize, hasError, or getErrors pay for one import-root mark and no nested promise registrations or descendant metadata. Trusted language data follows the compiler's single-owner/COW contract and uses the ordinary tree-shaped ref-index walk without identity, duplicate, active-path, or SCC bookkeeping.

    **Implementation sequencing.** Items 11 and 12 landed as two internally atomic parts of one design. The mirror-drain foundation includes one-stage `onValueResolve`, `onPromiseMirrorResolve` with `pendingConsumerCount`, drain-defined `settled` and single-commit semantics, `readLogicalProperty`, placement-sensitive counts, and migration of every mirror consumer. `verifyRefCounts` recounts a draining live mirror as `[1,0]` and reports a parents-graph cycle fatally. The imported-projection part adds `src/import.js`, lazy imported preparation, descendant marks, deterministic SCC cycle Errors, and cycle-aware normalize.

    **Import boundary.** `import(value, errorContext)` rejects every falsy context and returns `markImported(value, errorContext)`. This keeps truthiness a valid imported-provenance gate throughout the runtime; Cascada supplies an object and this sandbox may use a non-empty string. An object root receives the import/shared mark directly. A promise root keeps the one `onValueResolve` wrapper already provided by `markImported`, so its resolved root is marked before language consumers observe it. Import performs no graph walk, owns no `WeakSet` or worklist, and does not register nested promises.

    **Before ref-indexing.** The imported root's shared mark protects its complete subtree through inherited COW state. A mutation copies from that root before entering descendants; `shallowCopy` stamps reused children and flavors forked mirrors. `lookupPath` marks an extracted imported value, and later promise discovery receives the inherited import context. Internal aliases therefore remain isolated even when no counter operation is ever issued.

    **Imported preparation and ref-index commit.** Ordinary trusted ref-indexing remains a simple recursive tree walk. When that walk encounters a truthy own or inherited import context, it delegates the complete imported region to `src/import.js`. Imported preparation discovers the logical structure into one operation-local identity table. The first encounter creates one node record and expands that node; every owner/key placement is still recorded, including an edge whose target already has a record. This deduplicates imported-node computation without deduplicating structural edges, so aliases, parent multiplicity, and cycle-closing placements remain distinguishable. Active versus completed state distinguishes an imported DAG alias from a cycle. Extensibility creates no discovery mode; trusted nodes enter this identity table only when reached under imported provenance.

    Imported identity analysis is non-mutating and operates on this prepared structure. An already-ref-indexed node reached for the first time under imported provenance must still be inspected; existing counters are not proof that imported identity analysis or descendant marking has occurred. Issue 12 defines cycle Errors and their query, mutation, and normalize behavior; this step supplies the root-only import boundary and inherited context that identify the imported region. Only after preparation has staged its cycle Errors does `commitRefIndex` publish `SHARED` marks, counters, mirrors, and parent edges. It marks every tracked node retained under inherited imported provenance `SHARED`, but does not assign that context as the node's own import marker. Marking every retained node, rather than only duplicate identities, is required for cross-promise query deduplication: a promise may later expose an identity whose first occurrence was already traversed. Nodes carrying cycle Errors remain retained because raw bounded path access and normalization may still reach them. The prepared node records are reused for commit, so identity discovery and indexing do not reread the raw graph.

    **Imported DAG aliases.** After cycle Errors have been staged, derive imported aliases from the remaining projected placements. A cut cycle edge contributes neither alias multiplicity nor a reverse parent edge. Same-parent repeated keys, different parents, already-indexed islands, and aliases introduced by imported promise writeback all reuse one imported node record, while every projected structural edge still contributes to counts and exact parent-edge multiplicity. Counter-delta propagation and promise slots must never be identity-deduplicated. Trusted data has no equivalent alias-discovery pass; any reuse emitted by the language has already crossed lookup/shared ownership and is isolated by COW before it could create a duplicate identity inside one branch.

    **Promise settlement.** Nested promises remain untouched until path discovery, COW, or ref-indexing needs them. An import-flavored mirror marks its resolved branch root imported before live or revoked writeback prepares/ref-indexes it. The successful commit then marks that resolved branch's imported descendants shared before later FIFO query continuations inspect them. Root promises retain the import wrapper because no object root exists to carry context before they settle.

    **Attribution and non-extensible data.** Imported provenance remains inherited from the relevant boundary root until extraction, COW reuse, or promise settlement creates a new boundary that must carry it directly. Ref-index materialization records shared state without assigning every descendant a direct import boundary, so a shared descendant reachable from separately imported roots does not have one root's attribution context pinned onto every path. Non-extensible values are implicitly shared, store required preparation/counter/mirror state in WeakMap metadata, and otherwise follow ordinary indexing rules.

    Error queries keep one operation-local marked-node `WeakSet` across their captured promise tree. Other existing deduplication remains local to its semantics: imported preparation visits each imported node once, plain-copy normalization preserves imported DAG identity with an old-to-new map, and verification uses its own synchronous seen set. Do not introduce one generic deduplication abstraction across these different policies.

    Coverage must include: rejection of empty and other falsy import contexts; no descendant metadata or nested promise registration immediately after import; one wrapper for an imported promise root; mutation and extraction before ref-indexing; synchronous imported DAG aliases; an identity present synchronously and exposed again by a later promise; recursively exposed promise branches; second-edge cycle detection and exact multiplicity; non-extensible branches; and both metadata storage modes. Existing import tests that assert eager recursive marks or registrations must be replaced with these boundary and lazy-materialization assertions.

12. **Implemented: imported cycle Errors.** Imported host graphs remain physically unchanged. Each property-level `cycleError` is its attributed Error. Lazy imported preparation, mirror draining, atomic projected edge transitions, cycle-aware normalization, and both metadata storage modes implement the complete model below.

    **`src/import.js` boundary.** Both the public `import(value, errorContext)` operation and lazy imported-data preparation live in `src/import.js`; generic metadata storage and access remain in `src/meta.js`. The module exports only the public import operation and one internal `prepareImportedData` entry point; its discovery and SCC helpers are private. Import itself validates the truthy context and marks only the boundary root (or wraps a root promise), with no recursive work. `prepareImportedData` is called only when exact counters are required: the first ref-index build that crosses into imported provenance, an imported candidate entering an already ref-indexed edge/mirror, or reconstruction of an already ref-indexed COW copy. It is not called merely by ordinary import, lookup, path mutation, or COW of an unindexed value. It owns the operation-local identity table, imported DAG deduplication, active/completed states, write-target reachability, SCC analysis, and staged cycle-Error/shared state. It is pure: no values, metadata, mirrors, counters, or parent edges change until refcounts commits its result. This is the only step-12 file extraction; broader module separation remains issue 14.

    Trusted language data never enters imported preparation. The compiler gives each newly created value one owner; lookup marks an escaping value shared; mutation COWs before writing; and `a.prop = a` assigns the pre-mutation copy of `a`. Consequently a valid trusted branch contains neither duplicate tracked identities nor cycles. Its ref-index walk allocates no identity table, visited set, active-path state, SCC state, or write-target search. Raw kernel calls that violate this compiler contract are outside the runtime API rather than a separately checked fatal path.

    Import remains O(1), while lazy discovery, SCC analysis, and ref-index construction are recursive. The first counter-based operation on a very deep imported graph is therefore bounded by the JavaScript call stack; moving that cost to the first query is the deliberate lazy-import tradeoff.

    **Observable edge model.** Imported host data remains physically unchanged. A cyclic property keeps its raw value and receives a cached Error in `meta.cycleErrors[key]` or `mirror.cycleError`. Refcounting does not descend through that raw edge or install a reverse parent edge; the cycle Error contributes `[0,1]`, making the counter graph finite without changing the value graph. Lookup, mutation, and cycle-aware normalize continue through the raw logical value. `hasError` and `getErrors` expose the cached Error. The owner/key placement is the identity: aliases reaching the same property reuse one stable cached Error, while distinct cyclic properties receive distinct Errors.

    Every cached cycle Error message names its property key and carries the applicable import attribution, for example `Cyclic property "back" (imported at: ...)`. A full path is deliberately not stored because a shared property can have several paths. The first committed attribution for one owner/key wins, matching the existing first-import rule. Imported data is treated as externally immutable after crossing the boundary, so the cycle result may be cached; arbitrary later host writes are out of scope, while runtime promise settlement and language COW are managed transitions that must update it.

    **Deterministic imported-cycle Errors.** Cycle analysis consumes the operation-local imported structural records prepared by issue 11. Discovery interns each imported logical node once but records every owner/key edge; active and completed state are tracked independently of that identity table. Thus a completed-node edge is an imported DAG alias, while an active-node edge proves that cycle analysis is required. A plain visited-node skip must never discard the repeated placement.

    The active-path state is the acyclic fast path. Detecting an imported back-edge records that SCC analysis is needed but does not stop structural discovery; finish every reachable record and placement first. Then run Tarjan's algorithm or an equivalent O(V + E) SCC analysis over that complete table rather than walking or rereading the host graph again. Acyclic imports therefore allocate no SCC-specific state. For every cyclic SCC, including a one-node self-cycle, stage a cycle Error on every property whose owner and tracked target are both in that SCC. This root-independent set is the same no matter which branch is queried first; property enumeration order and earlier observations cannot change the number or identity of cyclic placements. Each such property gets its own stable attributed Error. Existing cycle Errors are cuts during later preparation and are not traversed again.

    A newly assigned or newly settled edge carrying imported provenance is handled incrementally. The same placement-aware rule applies when an initial trusted ref-index walk first reaches an imported child. First prepare its candidate value and any intrinsic imported SCCs without publishing them. If adding the virtual `parent[key]` edge would reach `parent` through the current counter graph, that new edge itself receives a cycle Error instead of a child parent-edge; existing cuts are respected, so a raw route that crosses a cycle cut does not create another counter cycle. The cut is published before prepared child indexes can re-enter that owner, and an older recursive frame discards its partial result if the back-reference completed the owner's index. This identifies the operation-created imported property directly without recomputing an otherwise valid indexed region. The resulting history distinction is intentional: importing a complete cyclic graph gives every intra-SCC property a cycle Error because there is no privileged closing edge, while an imported incremental transition gives one only to the edge known to have closed the cycle.

    **Preparation and transactionality.** Imported preparation must not skip a node merely because ordinary trusted ref-index metadata already exists on it; entering with imported provenance starts the imported identity analysis and descendant marking that trusted indexing deliberately omitted. Placement-specific write-target reachability is never cached as an object-wide fact. Existing cycle Errors and settled mirror values are part of the imported logical graph, not their stale physical properties. Preparation stages cycle Errors and mirror state operation-locally; one synchronous commit publishes the final value, cycle Error, counts, and parent edges.

    One edge-transition core owns this logic across ordinary replacement, deletion, cycle-Error publication, mirror writeback, and walk advances. `prepareEdgeTransition(owner, key, mirror, candidate)` prepares imported/shared state, a possible cycle Error, the index, and eventual counts without changing the live placement. Every live commit snapshots the old contribution, performs the already-validated placement update, swaps the reverse edge, and propagates exactly one delta from explicit next counts. A null mirror is an ordinary value transition. Assigning or copying a fresh Promise immediately commits its new mirror placement as `[1,0]`; the mirror's successful drain later uses the same primitive for `[1,0] -> final state`. A revoked mirror stores only private prepared state and never commits against its former parent.

    **Promise consumer ordering.** Collapse generic `onValueResolve` to the one-reaction form defined by issue 8, then route every callback that consumes a promise through a mirror — writeback, mutation/observation walks, forks, and error-query waits — through one `onPromiseMirrorResolve(mirror, fn)` wrapper. Share one private reaction runner so value conversion and fatal handling are not duplicated. The mirror wrapper increments `mirror.pendingConsumerCount` synchronously, registers directly on `mirror.promise` at that program position, and invokes `fn` with the same fulfilled-or-converted-Error value as `onValueResolve`. It owns one completion hook inside that reaction: the hook decrements after `fn` has run its synchronous body, including when rejection conversion or `fn` throws, and records a failed drain before propagating a fatal error. Runtime functions are not async: any deeper wait is registered before `fn` returns and owns its own mirror ordering, so the counter does not wait for a Promise returned by `fn`. A later rejection from that returned Promise is still fatal under issue 8, but it cannot retroactively roll back synchronous edge preparation that already completed. Generic promise uses that have no mirror continue to call `onValueResolve` directly.

    A mirror starts with `settled = false`; registering its mandatory writeback is its first counted consumer. Source resolution lets each successful consumer prepare and store the next `mirror.currentValue`/Error state before the following FIFO consumer runs, but the live edge remains operationally pending while any registered consumer remains. A successful decrement from one to zero is the only drain point: if no consumer failed synchronously, refresh the final preparation when the owner became ref-indexed after its previous preparation, synchronously commit that state to the live placement if the mirror is still live (or leave it private if revoked), and then set `settled = true`. This commit-time gate preserves downward closure and catches an imported cycle closed during the drain. A synchronous fatal completion, including a falsy thrown value, sets the mirror's failed-drain state, decrements bookkeeping, and leaves `settled = false` and the old live edge; later queued consumers may finish their private work but can never publish that failed drain. A consumer registered during the source-settlement-to-writeback gap therefore keeps the mirror pending until its advance has run. No program continuation, whether tied to the same promise or an unrelated one, can use the synchronous fast path before every earlier registered mirror consumer has applied its synchronous advance.

    **Logical reads and edge counts.** `settled` means the source resolved **and all registered mirror consumers drained**, not merely that the writeback callback ran; it also distinguishes a legitimate settled `undefined` from an uninitialized current value. Use two deliberately different operations:

    - `readLogicalProperty(parent, key)` checks the live mirror first. A settled mirror returns `currentValue` synchronously; an unsettled/draining mirror returns its original Promise regardless of the physical slot, forcing the caller through `onPromiseMirrorResolve` behind every earlier registration. With no mirror, it returns the own enumerable physical property. It ignores cycle metadata and is used by lookup, mutation, COW, promise forks, and `copyToPlainValue`; the own-enumerable physical read is its internal fallback rather than a second public operation.
    - `getPropertyRefCounts(parent, key)` returns `[1,0]` for an unresolved **or draining** live mirror. Only a settled mirror exposes its final prepared cycle Error/current child counts; without a mirror, any committed cycle Error returns `[0,1]`, a physical Promise `[1,0]`, and an ordinary value its normal counts. It is used by index construction, property replacement/deletion, COW counter construction, error collection, verification, and promise transitions. Keeping the draining edge at `[1,0]` is load-bearing: root-level normalize/hasError can answer directly from ancestor counters without calling `readLogicalProperty`, so a read-only guard would still expose a transient stale state.

    The parent contribution does not bounce through intermediate resolved values while consumers drain. It remains `[1,0]`, each consumer privately prepares the current value needed by the next consumer, and the zero-consumer commit performs the single `[1,0] -> final counts` delta. This prevents transient zero-crossings from waking normalize before an earlier-issued advance. If the mirror is overwritten or deleted first, removal detaches that one pending contribution; the old mirror continues preparing its captured private world without touching live counts.

    Do not call `getRefCounts(parent[key])` or inspect `parent[key]` directly where a mirror or cycle Error may change the counter contribution. A cycle cut is not followed by refcounting or given a reverse parent edge. The resulting counter graph is acyclic, so every ordinarily reachable node has exact `promiseCount`, `errorCount`, and parent maps; no cyclic counter propagation is needed. The raw value graph remains cyclic and is traversed only by finite path operations and cycle-aware normalization. Counter copying derives every edge contribution and parent edge in the copy's new world rather than cloning source totals or cycle Errors.

    **Non-extensible metadata.** Both storage modes have identical behavior. In inline-property mode, `META_MAP` is the fallback for non-extensible values that need import attribution, cycle Errors, mirrors, or logical counters; `metaOf` and `ensureMeta` are the only access boundary. An own inline metadata record is checked first and remains authoritative if host code makes its node non-extensible after metadata was attached. Non-extensible nodes use ordinary exact counters, and Promises and Errors inside them are valid language data.

    **Queries and normalize.** Observational path resolution retains the reached terminal placement `(owner, key, mirror?)` for counter-based callers. A terminal cycle Error answers the error queries, while normalize follows the raw logical target through its cycle-aware path. Path resolution itself ignores the Error: a longer path can deliberately cross a cyclic property and select a clean raw subpath, which is then evaluated from its own terminal placement. `lookupPath` continues to need only the raw logical value.

    Otherwise `hasError` needs no recursive cycle branch: the reached branch's `errorCount` includes every reachable ordinary Error and cached cycle Error. `getErrors` adds each cyclic property's cached Error once and never descends through that property. Errors, promises, or further cycles reachable only through its raw value are intentionally outside the query's projected branch; ordinary siblings are still collected and awaited. The operation-local Set deduplicates aliases of the same owner/key, while different cyclic properties remain different results. Both queries preserve their existing issue-time promise frontier and never pin or mutate data. Their coherence invariant compares the same exact path and projected terminal placement; it does not require an ancestor branch to report raw content hidden behind a cut edge.

    Error-query collection never mints a mirror. Ref-index preparation guarantees one already exists for every pending property in an indexed branch, so `collectErrorSearchWaits` uses a required-existing accessor and registers through `onPromiseMirrorResolve`. A missing mirror is a fatal downward-closure/writeback invariant failure; calling `getOrCreatePromiseMirror` there would hide the bug, register at the wrong FIFO position, and lose the holder policy captured at the legitimate birth site. A settled/drained mirror is consumed through its logical counts and is not re-registered merely because its holder still physically contains the Promise.

    Normalize keeps its existing counter-based settlement path for branches with no reachable cycle Error. An ordinary Error remains a normal Error result after counted promises drain. A draining mirror keeps `promiseCount > 0`, so normalize cannot return during the settlement/writeback gap or before an earlier registered advance. A clean subpath selected through raw lookup from inside the same imported graph has no reachable cycle Error and normalizes normally.

    No third subtree counter is added: every cycle Error deliberately contributes to ordinary `errorCount`. The rationale is not `hasError` (both designs keep it O(1)); it is that a permanent `cycleCount` would add storage, delta propagation, copy reconstruction, verification, and transition width to every indexed branch for a rare normalize classification that early exit makes cheap. At settlement, a cycle-aware counter-pruned traversal collapses immediately on an ordinary Error and otherwise records whether a cycle Error exists. Only the cycle-only case starts a raw logical traversal. That traversal owns one identity map, waits for recursively exposed raw promises, and constructs the topology-preserving plain copy in the same pass when requested; it never performs a second raw copy walk. The decision begins after the counted frontier has drained because an earlier-issued remainder may repair a cycle cut or replace an ordinary Error before zero. If repeated classification of a cycle-rich branch becomes measurable, a cached cycle-only bit is a compatible later optimization, but every descendant transition that can change the classification must invalidate it through the indexed parent graph; the counter schema is deliberately unchanged.

    The cycle-aware raw phase follows logical properties through cycle cuts and uses one identity map to preserve every DAG/cycle reference. It consumes existing mirrors at normalize's FIFO positions and recursively extends the raw promise frontier, including promises reachable only behind counter cuts; projected `promiseCount` alone is insufficient in this case. Ordinary Error values encountered by the raw walk still produce normalize's normal Error result after that frontier settles. With `plainCopy=false`, a cycle-only successful result is the original branch by identity. With `plainCopy=true`, the same traversal produces a metadata-free deep copy whose repeated and cyclic references point to their corresponding copies. A non-copy result remains language-internal logical data and may retain physical settled Promises in external or non-extensible holders; `plainCopy=true` remains the host/output boundary.

    The cycle-operation contract is therefore explicit: lookup and mutation traverse the raw imported data; normalize of an acyclic subpath behaves normally; normalize of a cycle-containing value preserves the raw cyclic topology by identity or in its plain copy; and `hasError`/`getErrors` report the cached per-property cycle Error. That Error is diagnostic rather than poisoning, so `hasError` may be true even though normalize succeeds. It cuts only refcount and error-query traversal, never the logical value seen by normalization.

    **Lookup, mutation, and COW.** Finite path lookup and mutation use logical mirror values but ignore cycle/Error metadata; they are safe on raw cycles because they never recursively scan the graph. Imported data is shared, so mutation COWs before its first language write and never changes the host object. COW remains path-copying rather than topology cloning: for `external.self = external`, assigning an unrelated root property produces a new root whose `self` still points to `external`, not to the copy.

    Cycle Errors are placement-specific and are never copied blindly to a new owner. Unchanged raw children retain metadata on their own nodes; every copied edge is recounted in its new placement. A cycle Error changes only when that exact logical owner/key placement transitions; changing a sibling edge does not trigger a global SCC rescan. Language mutation of imported data COWs, so copied placements are new and prepared without copied cycle Errors, while Errors on the physically unchanged external graph remain valid. Every missing copied key is defined as an own data property, so preserving an enumerable `__proto__` edge cannot invoke the legacy prototype setter. Replacing/deleting a cycle cut removes its `[0,1]` contribution only at commit. A mutation can repair or create a cycle cut, and later lookup, `hasError`, `getErrors`, normalize, and `verifyRefCounts` must all agree after unrelated COW, partial and complete repair, repeated cyclic path segments, deletion, and whole-boundary replacement.

    **Promise transitions.** The writeback consumer first prepares the source's resolved value as private mirror state; every following consumer prepares its advance on top of the preceding `mirror.currentValue`. These preparations ref-index the current candidate before it becomes available to the next FIFO consumer, but they neither attach it to the live parent nor change that parent's `[1,0]` contribution while the mirror drains. The final successful consumer performs one atomic live-edge commit only when the complete drain remained non-fatal. An extensible owned holder receives the final physical value only at this drain point, avoiding intermediate physical states; an external imported or non-extensible holder remains physically untouched.

    When the final promise placement closes a cycle, retain the raw prepared value in `mirror.currentValue`, store its attributed Error in `mirror.cycleError`, and commit the indexed parent directly from `[1,0]` to `[0,1]` without attaching a reverse edge to the raw target. Do not install either the Error or cycle-closing value into an external or non-extensible property; an extensible owned holder may store the raw final value because its counter graph remains cut by the Error. Intrinsic cyclic properties inside a resolved object receive `meta.cycleErrors` entries during preparation. A later prepared advance may repair or recreate the edge before drain, so only the final prepared Error/value state contributes to the live parent.

    A mirror attached to an external imported holder remains authoritative after a valid drain: retain the original Promise in that holder while logical operations use `mirror.currentValue`. Capture this external-holder placement at mirror birth from the walk's inherited import provenance; a nested external holder need not carry its own import mark under root-only marking. Keep that decision separate from the mirror's import context for attributing and marking its resolved value. A COW fork carries the context but belongs to a new language-owned holder and writes its final drained value physically only if that holder is still extensible, as do assigned and other ordinary language-owned mirrors. Consequently imported and non-extensible data remain physically unchanged without a promise exception.

    Live, revoked, and forked mirrors use the same prepared-edge state. Revoked preparation receives the owner, key, and mirror explicitly, records each private raw value plus any cycle Error without changing former parent counts, and remains available to operations that captured it; its successful drain sets `settled` but performs no attached-edge commit. A fork reads the source's prepared logical value at its FIFO position but prepares against the fork's own owner/key; an edge cyclic in the source need not be cyclic in the copy. Every later mirror advance goes through `prepareEdgeTransition`, never an unchecked `currentValue` assignment. Overwrite/delete clears the live mirror only during its own committed edge swap; already-registered consumers continue draining the revoked mirror privately.

    **Implemented cleanup addendum, slice 1: one prepared edge and cycle-Error representation.** Imported discovery records are the sole staged graph. Each node record owns one edge record per enumerable owner/key placement with the shape `{ key, value, mirror, cycleError }`: `value` is the captured logical value, `records.get(value)` identifies a prepared tracked target, `mirror` is the captured existing mirror when present, and `cycleError` is the effective committed or newly staged Error. Promise handling is derived from `value`, `mirror`, and the preparation's excluded mirror; there is no parallel target, promise-placement flag, or lookup table.

    One attributed cycle Error is stored directly in `meta.cycleErrors[key]`, `mirror.cycleError`, and prepared edge state. Presence means the projected edge is cut and contributes `[0,1]`; `hasError` and `getErrors` expose that Error, while normalize follows the raw cyclic data. Distinct cyclic properties still own distinct stable Error identities.

    Discovery captures an existing committed mark on its edge and treats it as a cut. Tarjan annotates unmarked intra-SCC edge records directly and reuses an existing mark object for the same placement. Commit compares the captured/staged mark with the currently committed mark by identity and publishes only a real change, preventing duplicate deltas and preserving first attribution. Imported commit iterates the same records, creates or updates required mirrors before publishing marks, and `commitRefIndex` consumes those records directly. Incremental reachability follows prepared records whenever present and falls back to the committed logical graph only for values outside that preparation, while respecting committed cuts in both cases. Aliases share one node record but retain every distinct edge record and exact multiplicity; no parallel placement collection or marker lookup remains.

    Cycle-Error storage deliberately retains two read views. `getCommittedCycleError(owner, key)` serves counters and the attached graph: a draining mirror remains `[1,0]`, so its private prepared Error is invisible until final commit. `getResolvedCycleError(placement)` serves the operation that captured a mirror and may observe that mirror's private prepared Error before drain. A mirrored placement stores its Error exclusively in `mirror.cycleError`; installing or removing the mirror keeps `meta.cycleErrors[key]` absent, so neither accessor needs to reconcile two stores. Do not merge these accessors or parameterize one ambiguous resolver. Low-level metadata needs one setter and clearer; both views return the same Error value.

    Preparation remains pure, transactional, and operation-local. To avoid a dependency cycle while ensuring that indexed cycle-Error publication uses the common bookkeeping path, its narrow commit boundary is `preparation.commit(commitCycleError)`: import owns record iteration, shared/imported metadata, and mirror creation, while the supplied synchronous callback publishes each changed cycle Error. In slice 1 that callback is backed by the existing committed cycle-Error bookkeeping; slice 2 reroutes its internals through the atomic live-edge commit without changing the boundary. An uncommitted or failed preparation publishes none of its staged descendant metadata, mirrors, cycle Errors, counters, or parent edges; the import boundary mark that caused preparation may already exist.

    **Implemented cleanup addendum, slice 2: one atomic live-edge commit.** Every live assignment, deletion, changed mark installation/removal, and successful mirror drain uses one internal synchronous bookkeeping primitive. It first snapshots the old effective counts and counted child, then performs the already-validated placement-specific value/mirror/mark update, removes the captured old reverse edge, attaches the explicitly supplied next child, and propagates exactly one delta from explicitly supplied next counts. It does not reread the next state from a draining mirror because `settled` flips only after commit. Expected property-shape failures occur before entry, so a failed placement update leaves counter bookkeeping untouched. Here atomic means one non-interleavable live transition and one count delta, not rollback from an internal fatal error.

    Preparation and revoked/private mirror updates remain outside the atomic primitive because they do not mutate a live indexed edge. An unindexed live owner may use the same entry point, which performs only its placement update because no counter graph is attached. Do not turn this into a generic graph framework: it owns only the invariant shared by attached owner/key transitions.

    The full inline and WeakMap suites cover a draining mirror whose private cycle Error is visible to its captured continuation but not to parent counts; replacement, reuse, clearing, and deletion of committed cycle Errors; and exact deltas through aliased parent-edge multiplicity. Superseded helpers, side tables, and paired fields are removed, and `README.md`, `initial-spec.md`, and `counters-implementation.md` describe direct `cycleErrors`. `mirror.currentValue` and prepared counts remain separate because merging them adds no further simplification.

13. **Implemented: uniform non-extensible node support.** The special frozen-subtree validation and counting model is removed. Frozen, sealed, and otherwise non-extensible objects are ordinary tracked nodes that are implicitly shared, store runtime metadata through the existing WeakMap fallback, and participate in the same lazy ref-index as extensible nodes. Import itself remains shallow; only the first counter-based operation on a branch pays to index its non-extensible nodes.

    Promise and Error values are valid own enumerable values inside non-extensible imported data. An existing own enumerable `__proto__` data property is also ordinary language data. Errors contribute ordinary error counts and every valid promise placement receives a normal mirror. Draining a mirror writes its final value physically only when `!mirror.externalHolder && Object.isExtensible(mirror.node)` at drain time. The dynamic check matters because a holder can become non-extensible after mirror creation. Otherwise the raw Promise remains in the holder and every runtime operation observes the settled logical value through the mirror. Explicit assign/delete operations continue to treat non-extensible objects as shared and COW before mutation, so neither language mutation nor promise settlement changes the original object.

    This uses no new META or mirror field beyond item 12's direct `cycleErrors`/`cycleError` representation. `hasSharedMark` derives sharedness for a non-extensible value, so `markShared` leaves it alone and sharing by itself allocates no META. A caller that also needs `importPrepared`, import context, counters, mirrors, or cycle Errors ensures META explicitly. Imported preparation uses one `ensureMeta(record.node)` and sets both `shared = true` and `importPrepared = true` on that already-required record. Other runtime needs use the WeakMap fallback.

    **One counter invariant.** After a successful `buildRefIndex`, every tracked value reached through the projected graph and every non-cut tracked child of an indexed node has a counter, regardless of extensibility. A cycle Error cuts its raw target from that graph, so the target is not required to have a counter merely because the owning placement is indexed. Ordinary counter consumers never cross the cut. Normalize alone may receive the raw target of a terminal/private cycle placement without a counter. `normalizeResolved` retains that one explicit no-counter branch, and `finishNormalize` skips counter-based classification only for that counterless terminal cycle. A counted terminal cycle still classifies first, so an ordinary Error is returned without waiting for promises reachable only through the raw cycle frontier. `classifyProjectedErrors` therefore receives only indexed roots and uses required counters throughout.

    Error queries, mirror continuations, normalize, verification, and live reverse-edge bookkeeping consume that invariant with required-counter checks. `addParentEdge` and `removeParentEdge` remain no-ops for untracked values but require a counter for every tracked child; `commitRefIndex` records each successfully indexed tracked child without a second optional check. A missing counter on an ordinary tracked edge is fatal downward-closure corruption, not a clean result. Promise observation always creates or reuses a mirror through `onPromiseMirrorResolve`, so terminal promise placements also carry mirrors. `getResolvedCycleError` keeps its committed-cycle-Error fallback for non-promise placements.

    Prepared imported records are indexed with one simple loop that calls ordinary `commitRefIndex(record.node, record.context, preparedRecords)` for every retained record. Existing-counter short-circuiting keeps the commit linear; the loop is not reduced to one root call because newly staged cycle cuts can disconnect prepared records from that root. A private draining closing cut retains its skip until final indexed drain preparation, when its attached state can be committed safely. Every resulting live transition continues through item 12's atomic edge commit.

    `Object.isExtensible` has only three semantic homes: metadata storage selection, implicit shared/COW handling, and the final mirror physical-write decision. The mutation walker has already COWed a non-extensible language parent before the synchronous target write. Own accessor, non-writable, non-enumerable, and non-configurable descriptor checks remain. Imported discovery, refcount calculation, observations, queries, normalize classification, property validation, and verification do not branch on extensibility.

    The tradeoff is deliberate: a queried non-extensible branch allocates WeakMap metadata, counters, mirrors, and reverse edges just like a queried mutable branch. This demand-driven cost buys one runtime model, preserves external identity, and avoids eager deep copying. External JavaScript can still mutate a mutable imported object; preventing that would require a separate deep-copy or deep-freeze boundary policy. Making only a language-owned property's descriptor non-writable/accessor-backed while leaving its holder extensible remains outside the language contract and may fail fatally; item 13 supports object-level non-extensibility, not arbitrary host descriptor mutation.

15. **Language integration.** The language layer must route external values and ownership-sensitive operations through the kernel entry points that establish Cascada's invariants.

    **Import.** Every incoming external value must pass through `import(value, errorContext)` before that value can become part of language-owned data. The language layer constructs the truthy error/attribution context (line, file, ...; a non-empty string in this sandbox) at the call site. Import marks the boundary root only; if that root is a promise, its derived import wrapper marks the resolved root before consumers run. Nested promises are discovered lazily by path operations, COW, or ref-indexing, all of which preserve the inherited context. This includes frozen/sealed external structures, external promises used inside language initializers, and extraction of branches from external results (`var x = getExternalValue().a` preserves the same context).

    Example lowering:

    ```js
    // Cascada source
    x = { a: getExternalPromise() }

    // Runtime shape
    x = { a: import(getExternalPromise(), "script.casc:12") }
    ```

    Once roots are routed through `import`, marker propagation is the kernel's job: `lookupPath` marks every escaping value from an imported region (including `sharedOwnership=false`), COW stamps reused children, and flavored mirrors mark settled values — the layer's whole duty is wrapping the boundary crossings and constructing contexts.

    **Output.** Values that remain inside Cascada may use `normalize(..., plainCopy=false)` and must continue to be accessed through runtime operations, because their logical settled state may live in mirrors while an external holder retains its physical Promise. Every boundary that hands ordinary data to host code, a renderer, serializer, or plugin uses `normalize(..., plainCopy=true)`. That copy materializes logical values, preserves DAG identity within the output, strips runtime metadata, and is the only promise-free physical-output contract.

    **Compiler single-owner rule.** The kernel cannot detect a raw unimported/unshared tracked value assigned to two locations. The compiler must guarantee it never emits that: escaping or RHS object values go through shared-ownership `lookupPath`. For `a.prop = a`, the RHS is evaluated first through that shared-ownership path, so `a.prop` receives a COW copy of `a` as it existed before `prop` was added. A raw kernel call like `assignPath(root, ["self"], root)` bypasses that lowering and is not valid compiler output.

    Issue 4 also created a class of fatal, compiler-facing throws that this layer must own — they are JS exceptions, never language Error values:

    - `import` with any falsy errorContext throws (an absent/empty context would silently disable provenance).
    - Any mutation of an owned own non-enumerable property throws. Assignment also rejects own accessors and non-writable data properties; deletion rejects own non-configurable properties.
    - Computed segments may contain `"__proto__"`. The kernel defines a missing key as an own enumerable data property, so neither this legacy accessor nor any other inherited setter participates in language writes.

    After this step, the kernel can trust language-created reuse to pass through ownership-marking boundaries. Imported descendants remain protected by inherited root sharing until extraction, COW, promise settlement, or ref-indexing materializes the direct marks it needs.

16. **Class-instance copy-on-write adapters.** Imported class instances are valid containers for Cascada-visible own enumerable data, but copying every non-array object into `{}` silently discards class identity and methods. Generic prototype preservation is not sound: `Object.create(Object.getPrototypeOf(value))` cannot reproduce private fields or native internal slots, and can therefore create an object that passes `instanceof` while its methods fail.

    Add an explicit host-only shallow-copy protocol, exposed as a Symbol that is outside the language-visible string-key graph. A participating class implements the protocol and returns a fresh, extensible instance with its private/non-enumerable/internal state copied and exactly the same own enumerable string keys and values as the source. The kernel validates that contract, then keeps ownership of its existing per-key work: shared/import marker propagation, promise-mirror forks, counter copying, and parent-edge registration. Plain objects, null-prototype records, and arrays retain their built-in copy paths; `normalize(..., plainCopy=true)` remains deliberately plain-data output.

    An unadapted non-plain object may still be observed and ref-indexed through its own enumerable data, but an operation that requires COW through it must produce an attributed language Error rather than silently demoting it to a plain object. A throwing adapter or one that returns the source, a non-extensible/untracked value, or a different enumerable key/value surface is a fatal host-contract violation and must leave no partial mirrors, counters, or writes.

    Tests must cover a nested imported class instance plus ordinary sibling and top-level properties; mutation must leave the external instance and siblings unchanged, preserve `instanceof`, and keep a method using private state functional. Also cover promise/Error fields through the adapted instance, shared aliases, an unadapted class, and malformed adapters under both metadata storage modes.
