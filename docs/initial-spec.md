I am implementing a programming language that uses copy-on-write for variable assignment and modification.
We support objects and arrays, the language is compiled into JavaScript.
But whenever possible we shall still reuse the structure.
When modifying only a child property - we shallow copies the object at each level along the modified path, iterating deeper only along the path and finally setting the changed property.


// Illustrative code to show shall-on-each-path-level copying
function simpleAssignPath(root, segments, value) {
  // No segments left: replace the whole value.
  if (segments.length === 0) {
    return value;
  }

  const [head, ...tail] = segments;

  if (root === null || typeof root !== "object") {
    return { [head]: simpleAssignPath(null, tail, value) };
  }

  const newChild = simpleAssignPath(root[head], tail, value);

  if (Array.isArray(root)) {
    const newRoot = [...root];
    newRoot[head] = newChild;
    return newRoot;
  }
  return { ...root, [head]: newChild };
}

We need to be selective when mutating the object. We use copy-on-write, but only when the object has moved outside of the variable (assigned to other vars, used as arguments, etc…).
var a = { x:1, y:2 }
a.x = 2 // no copy on write
var b = a;
a.x = 1 // this requires copy on write to avoid changing b

The same principle works selectively on branches

var location = { pos: {x:1,y:2}, delta = {x:3, y:4} }
var pos = location.pos
location.pos.x = 2 // copy on write of pos to avoid changing it
location.delta.x = 5 // no copy on write

Objects imported from outside calls also need not be modified, so copy on write is used on them too.

Core ownership and import contracts:

- Compiler-created tracked data is tree-shaped and singly owned. Every escape or reuse goes through shared-ownership lookup, and every later mutation through shared data uses copy-on-write. The lowering of `a.property = a` evaluates the right-hand side first, so it assigns a copy of `a` as it existed before `property` was added. Raw kernel calls that bypass this lowering are not valid compiler output.
- Every value entering from JavaScript passes through `import(value, errorContext)` with a truthy attribution context. Import marks only the boundary root, or wraps a root promise so its resolved root is marked before language consumers run. It performs no descendant walk, cycle search, counter work, mirror discovery, or nested-promise registration.
- Imported descendants inherit the boundary's shared/imported state while a path is traversed. Their graph may contain aliases and cycles. The runtime discovers and prepares those identities only when exact subtree counters are first required, and only imported provenance pays for identity, active-path, or strongly-connected-component analysis.
- Imported host data is physically immutable from Cascada's point of view. Runtime path mutation first copies it. Promise settlement and cycle detection do not replace properties on the host object; logical settled values and cycle diagnostics live in runtime metadata.
- A cyclic imported property keeps its raw logical value and has one stable attributed cycle Error associated with that owner/key placement. The counter graph cuts that edge and counts it as `[0,1]`, with no reverse parent edge. This cycle Error is diagnostic, not a poisoning value in the logical graph: hasError/getErrors expose it, while lookup and mutation may follow the raw value and normalize may successfully preserve the cycle.
- Operations describe the state at their own program position. Their synchronous prefixes run in issue order and promise continuations register in that same order. Later operations may change the Chain, but they cannot change the value observed by an earlier lookup, error query, or normalize call.

Additionally, promises are handled transparently. A promise mirror holds the logical value of a promise-valued property and orders every operation waiting on it. Once the source has resolved and all consumers already registered on that mirror have drained, an extensible language-owned holder receives the final logical value; an imported or non-extensible holder keeps its original Promise property and the mirror remains authoritative. If the promise rejects, the settlement boundary converts the rejection to the language error value. Cascada's real error value is a special immediately-rejecting thenable, so rejecting promises are valid error values too; this sandbox often uses JavaScript `Error` objects to stand in for that value. Below, `Error` means this language-level error value unless JavaScript `Error` is mentioned explicitly. Also assignPath can receive an error value as value and it has to be properly referenced (see Subtree counters).

Fatal runtime failures are distinct from language Error values: they are reported once per object identity and the original value is thrown. A fatal failure in a discarded internal mutator continuation therefore remains a host-unhandled rejection after reporting; mutators still return nothing. Only rejection of a data promise at a value-settlement boundary is converted to a language Error value.

Public root operations take a `Chain`, not a bare root value. The Chain owns a
private `_state.value` holder slot that is outside the language graph; operators
run the value-root algorithm through that holder, so root promises use the normal
promise mirror machinery on the holder's `value` key. Other Chain fields are never
counted, copied, marked, mirrored, or validated as language data.

We have these operations/helpers:
Path resolution uses `walkObservationPath(chain, path, onResolved)`. The final path segment is the optional target (or an empty path targets the root), while every preceding segment is required. Existing Errors are propagated; trying to dereference a missing/null/undefined or primitive required value produces a language Error. The COW mutation walker receives the same complete target path but remains separate because it installs copied branches while traversing.
import(obj, errorContext) - requires a truthy attribution context, marks only the outside value's boundary root imported/shared, and returns without waiting. A non-promise root is returned synchronously; a promise root returns one derived promise that marks its resolved root before language consumers run. Import performs no descendant walk, counter work, mirror discovery, nested-promise registration, or property writeback. Descendant provenance is inherited until lazy imported preparation, extraction, COW, or promise settlement needs a direct mark. Imported Error values remain language errors and count for hasError/getErrors/normalize. External objects are not expected to be mutated by outside code after import.
assignPath(chain, segments, value) - for changing the whole object or any path value. It mutates the Chain state and returns nothing; values are read only through lookupPath, normalize, hasError, or getErrors. Any branch/property with the shared mark will be copied-on-write. Mirrors are created only by ordinary assignment, walk discovery, COW forking, or ref-indexing. The final property is the optional mutation target and may be added or replaced; every preceding value must exist and be trackable. A missing/null/undefined or primitive intermediate is replaced by a language Error instead of being autovivified. Path segments are objects, properties are assigned. The language-visible object surface is own enumerable string keys. Inherited properties remain hidden, and every missing key, including `__proto__`, is created as an own enumerable data property without invoking inherited setters. Owned non-enumerable properties throw, while shared/imported branches COW first and shadow non-enumerables as missing. An Error at the target key is replaceable; an Error used as an intermediate path node is propagated and not entered.
deletePath(chain, segments) - for deleting a path. It mutates the Chain state and returns nothing. The final property may be absent, in which case deletion is a no-op; every preceding value must exist and be trackable, with a broken prefix replaced by a language Error. An empty path replaces the whole root with null. Array element deletion deletes the own index property and preserves array length, so later reads see it as missing/undefined. An Error at the target key is deletable; an Error used as an intermediate path node is propagated and not entered.
lookupPath(chain, segments, sharedOwnership=true) - returns the value. The final target may be missing and then returns undefined; a missing/null/undefined or primitive intermediate returns a language Error because another property was accessed through it. By default a returned tracked value is marked as shared, because it is now shared between its owning variable/path and the caller; because of this even root access has to go through lookupPath. If sharedOwnership is false, the lookup does not create shared ownership, so the returned object/array is not marked. This is valid for pure read/inspection, or when ownership is ceded to the caller as the last use of the original owner.
normalize(chain, segments, sharedOwnership=true, plainCopy=false) - resolves the path with lookupPath's path rules, then returns the branch's state at normalize's program position after every promise in that captured logical branch has settled. It marks exactly what escapes, plus any branch it must pin while waiting; later operations therefore COW away while earlier registered remainders may still finish into the pinned world. Ordinary Errors collapse the result to one Error in this sandbox. Imported cycle markers do not replace their values: normalize follows the raw logical graph, waits for promises even behind counter cuts, and preserves cycles and aliases. With plainCopy=false a cycle-only success returns the settled branch by identity; with plainCopy=true it returns a metadata-free deep copy that reconstructs the same topology and materializes logical mirror values. A result already decided at issue time is synchronous; otherwise normalize returns a promise.
hasError(chain, segments) - whether the property is an error, or there was an error along the path. A missing final target is false; a missing/null/undefined or primitive intermediate becomes a language Error and is true, while an existing Error is propagated. At the reached branch, errorCount > 0 is true immediately; promiseCount === 0 is false immediately; otherwise it follows only the pending promises (descending only nodes with promiseCount > 0) and probes again at each settlement - first error wins, false once everything settles. A pure query: it never marks anything. The wait set is captured at issue time and extended only through resolved values read at hasError's own FIFO slots, so the answer is exact for the state when hasError was issued; later-issued installs are not observed, and later overwrite/delete of a watched pending key does not change the answer, though it may disconnect the eventual resolved value from the live tree.
getErrors(chain, segments) - returns the distinct Error identities reachable in the issue-time branch. It resolves the full path observationally: an existing Error along the path is propagated, while a broken intermediate produces a path-access Error. Any successfully reached terminal with no reachable Errors returns an empty array, including missing, null/undefined, primitive, and clean tracked terminals regardless of extensibility. The reached value is ref-indexed once. It follows the captured promise frontier recursively and waits for all of it, because the complete result cannot be known early. Counter-guided walking skips clean regions, one call-local Set deduplicates Error identities, and no shared mark or normalize settlement pin is created. A settled result is returned synchronously; otherwise a promise resolves to the final array after every captured and recursively exposed promise settles.

All operations run their synchronous prefix immediately, including hasError, getErrors, and normalize. assignPath and deletePath return nothing; an observation returns a direct answer when it is already decided at that program position and otherwise returns a promise. Calls on one Chain are issued sequentially but are not awaited between calls: each prefix commits its synchronous work and registers every required continuation before the next operation begins. Consumers of one promise mirror register directly on the same raw promise in issue order and drain in FIFO order. A mirror is not synchronously readable while any registered consumer remains, which prevents a later operation from overtaking earlier work during the settlement-to-writeback window.

Sharing is evaluated only from the point of the target language. Promises and resolved values are interchangeable, and runtime marks are invisible. The shared mark lives in the per-node metadata record (see Subtree counters - Metadata record). Every non-extensible object is implicitly shared; when it needs attribution, counters, mirrors, or logical edge metadata, that record is stored in the WeakMap fallback rather than on the object.

A promise exposed through shared-ownership lookupPath/import carries the same SHARED obligation as a directly returned object/array. If it resolves to an object/array, the resolved value is marked SHARED before consumers can observe it.

Shallow Copy - a COW copy duplicates the language-visible enumerable string keys into a fresh object or array and preserves array length. The copy is a new owned/unmarked node; the source keeps its shared mark. Reused non-path tracked children are marked shared, while the current path child is protected by the walk's inherited shared state until it is replaced or copied. The copy receives a fresh metadata record, eagerly forked promise mirrors, and, when the source is indexed, counters and parent edges reconstructed from each copied logical placement. Cycle markers are placement-specific and are never copied blindly. Imported provenance is propagated to reused children and forked mirrors. Each missing key is defined as an own enumerable data property, so copying `__proto__` cannot invoke the prototype setter. The scan is O(width of the node), the same cost class as the copy itself.

Shared Marking (requiring copy-on-write) - Import and shared-ownership lookupPath create shared values. Import marks only its boundary root immediately. Its descendants are logically inside that shared/imported branch; mutation walks track inherited state, COW stamps reused children, extracted values receive direct marks, and lazy imported preparation marks every retained imported node before counter-based queries traverse it. Import-flavored mirrors preserve the inherited context through settlement. A shared-ownership lookup marks the returned branch root. Non-extensible objects are implicitly shared; sharing alone allocates no metadata for them, while attribution, counters, mirrors, or logical edge state use the WeakMap fallback. All operations that pass, return, or store a whole existing object must use sharedOwnership=true unless they can prove ownership transfer or pure inspection.

Value resolve - each node can have a hidden promise mirror map in its metadata record (see Subtree counters - Metadata record). A mirror stores the exact promise, its newest logical `currentValue`, whether the source has resolved, how many registered mirror consumers remain, and any placement-specific cycle Error. The mandatory writeback is the first counted consumer. Every operation waiting on the mirror registers directly on the same raw promise and increments the pending count synchronously; consumers then prepare `currentValue` in FIFO order. The mirror becomes settled only after the source has resolved and all registered consumers have drained. Until then, logical reads return/register through the original promise rather than synchronously exposing a stale intermediate value. A non-fatal drain commits exactly one final live edge transition. If the owner acquired a ref index after the latest private preparation, the drain first refreshes that preparation against the now-indexed owner, preserving downward closure and detecting a cycle closed during the gap. An extensible language-owned holder receives that final value physically; an external imported or non-extensible holder keeps its Promise and uses the mirror as the logical property. Revoked mirrors drain privately for operations that already captured them and never change the former parent. A normal owned resolution remains owned; import, lookup escape, and non-path forks apply shared/imported marking at their respective ownership boundaries. Every synchronous overwrite/delete clears the live mirror entry even when assigning the same promise. All required accessible promises start resolving immediately: use `.then` and continue walking rather than await and enter, preserving registration order while unrelated branches proceed in parallel.

A promise mirror is born at exactly three points, and where its starting `currentValue` comes from differs:

- Assignment: a promise is assigned to a property (assignPath value, literal initialization). A fresh promise mirror is always created, even if the same promise instance was already at this key - two assignments are divergent worlds and must not share `currentValue`. The writeback continuation is registered immediately at assignment, before any operation can register on that promise for this key, so the writeback is always the first responder and `currentValue` starts as the raw resolved value.

- Discovery: a walk or operation reaches a promise key that has no mirror (imported data, raw literals - any orphan). The mirror is created and its writeback registered at the discovering operation's program position; `currentValue` starts as the raw resolved value. A mirror discovered while a lookup or ref-indexing walk carries import context records that context, and its writeback marks the settled value imported+shared before any consumer can observe it - settled external data is a fresh boundary crossing.

- Fork: a copy-on-write shallow copy duplicates a key whose logical value is still represented by a promise mirror. The copy's mirror is created eagerly at copy time. Its first consumer reads the source mirror's `currentValue` at the copier's FIFO slot, so it includes every earlier advance and no later one; it does not reseed from the raw promise result. The copy is a language-owned holder, even when the fork carries imported attribution, so its own final non-fatal drain may write the prepared value physically while the holder remains extensible. Non-path captures are marked shared because both worlds initially reuse them; a current-path capture relies on the walk's inherited shared state. After the fork, source and copy mirrors advance independently through the same counted-consumer and final-commit rules.

assignPath - receives the complete target path: the final segment is the optional target property, every preceding segment is required, and an empty path targets the root. If it reaches a promise before the target, it registers through that property's mirror at assignPath's program position and continues later from the mirror's prepared logical value. If a required intermediate is missing/null/undefined or primitive, it replaces that intermediate with a path-access Error and stops; there is no autovivification. At the target it assigns or creates the property. A shared or non-extensible branch is copied along the path; non-enumerable properties are not copied, so writes shadow them as missing. Missing own properties, including `__proto__`, are defined as enumerable writable configurable data properties, safely shadowing inherited accessors and non-writable values. Mutating an owned own non-enumerable property, accessor, or non-writable property throws before edge preparation. An intermediate Error is propagated unchanged, while an Error at the target is replaceable. The Chain's `_state.value` slot is the authoritative root location: copied/replaced roots are installed there synchronously, while a pending root remains represented by its mirror until the mirror's final live transition. assignPath returns nothing.

lookupPath - returns the value it reaches. It reads only own enumerable properties, including an existing own enumerable `__proto__` data property; inherited and own non-enumerable properties are treated as missing. If the reached value is a promise, or if a promise lies along the path, it returns a promise that resolves to the value once reachable - continuing through `currentValue` if the path was mutated during resolution (see Value resolve), never the raw resolved value. The returned promise is not awaited by lookupPath itself; values resolve only at the point of consumption. If sharedOwnership is true, marks the return shared if it is an array or object; if the return is a promise, it ensures the resolved value is marked shared. sharedOwnership=false lookups are for inspection or final ownership transfer and do not share ownership. Path and Error handling otherwise follows assignPath. Any Error or rejecting promise along the path results in an Error return.

deletePath - resolves every required parent-path segment (a segment may be a promise, continued through `currentValue` per Value resolve, not awaited), then deletes the optional target property, clearing its promise mirror so no later writeback re-mirrors it. A missing target, including inherited `__proto__`, is a no-op; a missing/null/undefined or primitive intermediate is replaced by a path-access Error. COWs shared branches along the way (the COW approach above), shadowing non-enumerables as missing after the copy. Array targets are normal property deletes: the index property is removed and length is preserved. Deleting an owned own non-enumerable or non-configurable property is fatal. These checks happen before edge preparation or bookkeeping. Any existing intermediate Error is propagated unchanged, while an Error target can be deleted. deletePath returns nothing; the Chain remains the authoritative root location.

Optimizations:

Subtree counters (implementation - supersedes the CLEAN flag, and settles the RefCounting question: we count promise/error content, not references; ownership stays mark-based). Every ref-indexed node carries exact counts of the pending promises and Error values reachable in its subtree (ref-indexing is lazy and branch-level - see below). hasError becomes O(1) at a settled node, and normalize's completion becomes an event - the watched promiseCount reaching 0 - instead of repeated rescanning. CLEAN survives only as a derived property: clean ⇔ promiseCount + errorCount === 0. Counters answer "what is pending/broken below me"; the SHARED mark answers "who else can see me" - the two never substitute for each other.

Metadata record - all runtime metadata lives in one per-node record. `STORE_META_IN_WEAKMAP` selects WeakMap storage; inline mode uses one own non-enumerable Symbol when the node is extensible and the same WeakMap as a fallback when it is not. Inline lookup is own-property-only and an existing inline record remains authoritative if its node later becomes non-extensible, so metadata is neither inherited nor lost. The record is invisible to the language and never copied wholesale by COW or plain normalization. Record existence does not imply live counters: mirrors, shared/import marks, and placement Errors may exist before a branch is ref-indexed. Both storage modes must have identical behavior and run the complete test matrix.
- shared - the SHARED mark; set once, never cleared; false at birth is the old "no mark". hasSharedMark(v) = meta(v)?.shared === true || !Object.isExtensible(v).
- mirrors - the promise mirror map (see Value resolve), lazy.
- promiseCount, errorCount - the counters, valid only when the node is ref-indexed. They describe the projected logical graph, not blindly the physical properties: an unresolved or draining promise placement contributes `[1,0]`, a cycle marker contributes `[0,1]`, and an ordinary tracked child contributes its totals once per structural edge.
- parents - exact reverse edges and the ref-indexed marker. `parents === undefined` means counters are not live; an empty `Map` means ref-indexed root / no ref-indexed parents; a populated `Map` is `Map<parentNode, edgeCount>` (a parent referencing the same child under two keys has multiplicity 2). Every commit that creates or removes a parent/child edge updates it; when an edge count reaches 0, delete that parent entry, never the `parents` map itself. References are strong in the sandbox (a world held by a suspended reader must keep receiving deltas); WeakRef edges pruned on dead deref are required in the real implementation, sound because liveness then coincides with observability - with strong edges every COW'd-away world stays referenced by its reused children and keeps receiving deltas, so per-delta cost grows with the COW history of a hot key: a time tax, not just memory.
- settlementPromise - optional shared promise generation added only by normalize while callers are waiting for promiseCount to verify at 0; hasError uses no settlement wait - its pending handling is the per-call Promise.all tree returned by probeIndexedBranchForErrors, not META fields.
- settlementResolve - resolver for settlementPromise.
- settlementVerifyScheduled - boolean latch that coalesces settlement verification callbacks while settlementPromise exists.
- importContext - the direct import-boundary marker and attribution: undefined means no direct boundary; otherwise it holds the required truthy context, with first import winning. Traversal also carries inherited import context, so descendants need no eager direct marker. Extraction, COW reuse, and promise settlement create a direct boundary only where later independent use requires one. Setting it also sets the shared mark. Non-extensible nodes use the metadata WeakMap fallback in either configured storage mode.
- importPrepared - the internal cache that records completion of lazy imported-graph preparation for this boundary; ordinary import does not set it.
- edgeMarks - optional owner/key map whose values are attributed cycle Errors. Each mark cuts that raw edge from the projected counter graph and contributes `[0,1]`; path operations and cycle-aware normalize may still follow the raw value, while error queries expose the mark itself.

Non-extensible rule: frozen, sealed, and otherwise non-extensible objects are ordinary tracked nodes and may contain Promises or Error values. They are implicitly shared, so language mutation COWs before writing. Ref-indexing gives every reached tracked node normal counters in the WeakMap metadata fallback, and every promise placement has a normal mirror. A mirror writes its final value physically only while its language-owned holder remains extensible; otherwise the raw Promise stays in the property and all runtime operations read the logical settled value through the mirror.

getRefCounts(value): a raw promise is `[1,0]`, an Error is `[0,1]`, a primitive is `[0,0]`, and a tracked value must be ref-indexed and uses its stored totals. Placement-sensitive code must instead use `getPropertyRefCounts(owner, key)`, which consults the live mirror and committed edge mark before the raw property. A missing child counter beneath an indexed ordinary edge is a fatal downward-closure violation rather than an invitation to index implicitly. Counts change only at committed edge transitions in the proper FIFO slot.

applyCountDelta(node, dPromise, dError): add to the node's counters, then recurse into every parent multiplied by edge count; zero deltas short-circuit. Trusted data is tree-shaped, imported DAG aliases are represented by exact edge multiplicity, and imported cycle/Error placements cut the projected graph, so parent propagation is always acyclic.

Lazy, branch-level ref-indexing - counter metadata exists only where normalize, hasError, or getErrors has been used; uncounted data pays no counter or imported-graph preparation cost. The first such query ref-indexes exactly its reached branch at the caller's program position. Ref-indexing stops at compatible already-indexed sub-branches, connects their boundary edge, and maintains the resulting region forever. Indexed regions are downward-closed through every ordinary non-promise edge: a value entering an indexed parent is fully prepared before commit, and promise writeback prepares the resolved child before query continuations inspect it. The gate is checked at commit time, not captured when a continuation is registered. Copies of indexed nodes reconstruct their counters and placement edges from their copied logical properties.

Ref-indexing / imported graph preparation - trusted language data uses a direct tree walk. Once inherited import context is active, ref-indexing delegates the imported region to lazy imported preparation. That preparation deduplicates node discovery while retaining every owner/key edge, distinguishes DAG aliases from active cycles, and deterministically identifies cyclic strongly connected components. It stages shared marks, promise mirrors, per-placement cycle Errors, counters, and reverse edges without mutating host data. Only a complete preparation is committed. Cyclic placements keep their raw values but contribute `[0,1]`. Existing indexed nodes must be revisited when first reached under imported provenance because trusted indexing is not proof of imported graph preparation.

Commit points - every edge change is prepared against the old placement and then committed once. Preparation derives the next raw logical value, mirror state, cycle marker, child counts, and reverse edges without publishing anything. Commit removes the old effective contribution and installs the new physical value when the holder policy allows it, marker, mirror, counters, and parent edges as one synchronous transition. Delete uses the same old-edge removal without replacement preparation. Fatal preparation leaves the previous graph unchanged. Mirror lifecycle remains in the mirror layer; low-level count propagation remains in refcounts. The caller sites:
assignPath target - overwrite; a newly assigned pending promise contributes [1,0], its mirror registered at assignment as before.
deletePath target - subtract the removed value's counts, remove the edge.
Promise settlement/writeback - the writeback and every registered mirror consumer prepare successive private logical values in FIFO order while the attached placement remains `[1,0]`. After the final non-fatal consumer drains, a live mirror performs one edge commit from the promise to the final prepared value or marker; a revoked mirror publishes only its private settled state. Imported host holders and holders that became non-extensible retain their physical Promise after the logical transition. Query continuations consume this already-prepared mirror state and never perform fallback ref-indexing.
Walk installs - the synchronous next !== child replacement and the suspended parent[key] = next, same rule.
shallowCopy - after copying language-visible keys and forking promise mirrors, counter reconstruction is a no-op unless the source is ref-indexed. An indexed copy starts with an empty parent map and derives every copied edge's effective contribution and reverse child edge from its own logical placement; it does not clone source totals or placement markers.

normalize(chain, segments, sharedOwnership, plainCopy) - observational path resolution threads import context and consumes logical mirror values. An Error or untracked terminal is returned directly. Otherwise normalize ref-indexes the reached branch at its program position and marks exactly what escapes, plus a branch that must be pinned while pending. A normal acyclic branch waits on the shared settlement generation until `promiseCount` remains zero after the mirror-consumer FIFO drains. At settlement, a counter-pruned classification returns immediately upon finding an ordinary Error. If it finds only cycle markers, one raw logical traversal follows the values behind those cuts, uses an identity map to terminate and preserve topology, and recursively extends its promise frontier. With plainCopy=true, that same raw traversal constructs the complete metadata-free copy; it does not run a second copy walk. A cycle-only success returns the original branch when plainCopy=false and the topology-preserving copy when plainCopy=true. Host-facing boundaries must use plainCopy=true because an imported holder's settled logical value may exist only in its mirror.
hasError(chain, segments) - observational path resolution treats a missing final target as false and a broken required prefix or existing Error as true. At a tracked terminal it builds the generic ref index exactly once, at the path-value boundary. `errorCount > 0` answers true immediately, including cycle markers; `promiseCount === 0` answers false. Otherwise the query follows only the captured pending mirror frontier. Each wait runs through the required existing mirror after writeback has prepared and indexed its logical value, extends the nested wait tree, and may resolve the operation's first-error race. hasError never marks or pins data. Later-issued installs are outside its captured state, while overwrite/delete cannot erase the private result of a mirror already captured at hasError's own FIFO slot.

Error-query deduplication is operation-local and precise. One `visited` WeakSet spans recursive promise slots, and only nodes with shared status are checked and added. Lazy imported preparation marks every retained imported node before query traversal, so imported aliases are visited once across synchronous and asynchronous branches. Trusted unmarked data is singly owned and cannot recur. A cycle-marked placement is collected as one Error and cut before traversal could follow its raw back-edge.

getErrors(chain, segments) - resolves the full path observationally, builds the generic ref index only at the reached path value, and returns each reachable Error identity once. Missing, null/undefined, primitive, and clean tracked terminals return an empty array. An Error along the path or a generated path-access Error becomes the sole result. The shared query walk uses counters to skip clean regions, collects ordinary Errors plus cycle Errors into one call-local Set, and never descends through a marked placement. Every captured mirror wait consumes the already-prepared `currentValue` and returns its nested readiness tree, so getErrors waits for the complete issue-time promise frontier before materializing the Set. It never marks, pins, races for an early answer, mints a query-time mirror, or calls buildRefIndex behind promise boundaries.

Correctness:

Cycles: the recursive counter graph can never contain cycles, although imported raw values may.
a.property.subproperty = a
The compiler prevents trusted cycles through ownership marking and COW. Imported graphs are inspected lazily when counters first need them. Preparation records every imported owner/key edge, distinguishes aliases from cycles, and places a stable attributed cycle Error on each chosen cyclic property without replacing its raw value. That placement contributes `[0,1]`, is not followed by counters, and has no reverse parent edge. Promise settlement can create or repair the same placement state through its mirror. This projected acyclic graph makes delta propagation terminate while finite lookup/mutation and cycle-aware normalize retain access to the raw topology.

Object reuse: An assignment or initializer targets one variable, so `var x = {a:1, b:2}` and an async value assigned to one variable are owned and need no mark. `var y = x` obtains x through shared-ownership lookup and marks the reused value; pure inspection or final ownership transfer uses sharedOwnership=false. Import marks only its root immediately. When counters later prepare an imported region, they deduplicate node discovery, preserve every structural edge and parent multiplicity, and mark every retained imported node shared before error-query traversal.

Race conditions: with assignPath and deletePath, it may look as if we are opening the possibility for race conditions. While one operation awaits a promise that resolves some data, a subsequent operation may modify the data - or commit synchronously and overtake the suspended one entirely. Why neither can corrupt anything or diverge from sequential semantics:

Program order = FIFO order. Operations are issued in program order and register their continuations at their own program position; JS runs continuations on one promise in registration order. So all consumers of one promise mirror resume in program order, each advancing the mirror's `currentValue` on top of the previous operation's version (V → V′ → V″). Immediate resolution of all accessible promises is required for this to hold (use .then and keep walking, never await + enter).

Owned promise writeback does not mark by itself. Escape marking (shared-ownership lookupPath/import) and fork marking are what force advances to copy instead of mutating a value that another consumer or forked world still references.

Suspended reads are immune to overtaking writes: a suspended lookupPath continues through the mirror's private `currentValue`, never the live property. A later operation that synchronously reassigns or deletes the key (clearPromiseMirror + assignment) is invisible to it - the read resolves to the mirror value as of its own program position, exactly what sequential execution would have returned.

Suspended writes are fenced by the promise mirror map: a resumed write mirrors into the live tree only if its mirror still owns the key (map[key] === mirror - revoked by a later assignment or clearPromiseMirror). A revoked write completes on the disconnected mirror value - a wasted copy, never corruption - matching sequential semantics, where the later operation's assignment or delete overwrote that write anyway. In particular, a delete followed by a resumed write cannot recreate the deleted path.

Reassignment or delete at an ancestor orphans the whole subtree instead: local mirrors inside it may still succeed (the guard is deliberately local), but every effect is confined to the disconnected branch - a waste, not a validity issue - and the live tree matches sequential semantics, where the ancestor operation discarded that branch anyway.

COW forks of keys holding promises cannot reorder anything either: the copy's mirror is seeded from the source mirror's `currentValue` at the copier's own FIFO slot and the captured value is marked shared, so the copied world contains exactly the writes of operations issued before the copy, and operations issued after advance the two mirrors independently through copy-on-write (see Value resolve - Fork).
