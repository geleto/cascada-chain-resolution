I am implementing a programming language that uses copy-on-write for variable assignment and modification.
We support objects and arrays, the language is compiled into JavaScript.
But whenever possible we shall still reuse the structure.
When modifying only a child property - we shallow copies the object at each level along the modified path, iterating deeper only along the path and finally setting the changed property.


// Illustrative code to show shall-on-each-path-level copying
function simpleAssignPath(root, segments, value) {
  // No segments left: replace the whole value.
  if (segments.length === 0) {
    return value;
  }

  const [head, ...tail] = segments;

  if (root === null || typeof root !== "object") {
    return { [head]: simpleAssignPath(null, tail, value) };
  }

  const newChild = simpleAssignPath(root[head], tail, value);

  if (Array.isArray(root)) {
    const newRoot = [...root];
    newRoot[head] = newChild;
    return newRoot;
  }
  return { ...root, [head]: newChild };
}

We need to be selective when mutating the object. We use copy-on-write, but only when the object has moved outside of the variable (assigned to other vars, used as arguments, etc…).
var a = { x:1, y:2 }
a.x = 2 // no copy on write
var b = a;
a.x = 1 // this requires copy on write to avoid changing b

The same principle works selectively on branches

var location = { pos: {x:1,y:2}, delta = {x:3, y:4} }
var pos = location.pos
location.pos.x = 2 // copy on write of pos to avoid changing it
location.delta.x = 5 // no copy on write

Objects imported from outside calls also need not be modified, so copy on write is used on them too.

Additionally, promises are handled transparently. If a property is set to a promise, once that promise is resolved, it is replaced by the resolved value automatically. If the value rejects, it is replaced by the language error value. Cascada's real error value is a special immediately-rejecting thenable, so rejecting promises are valid error values too; this sandbox often uses JavaScript `Error` objects to stand in for that value. Below, `Error` means this language-level error value unless JavaScript `Error` is mentioned explicitly. Also assignPath can receive an error value as value and it has to be properly referenced(see Subtree counters). Any operation or walk that encounters a settled-but-unreplaced promise replaces it in place (with counter propagation), and registers writeback for pending unregistered ones it discovers. The one exception is a promise key on a non-extensible holder: nothing can ever be written there, so reads resolve it mirror-free through the raw settled value (see Frozen rule).

Fatal runtime failures are distinct from language Error values: they are reported once per object identity and the original value is thrown. A fatal failure in a discarded internal mutator continuation therefore remains a host-unhandled rejection after reporting; mutators still return nothing. Only rejection of a data promise at a value-settlement boundary is converted to a language Error value.

Public root operations take a `Chain`, not a bare root value. The Chain owns a
private `_state.value` holder slot that is outside the language graph; operators
run the value-root algorithm through that holder, so root promises use the normal
promise mirror machinery on the holder's `value` key. Other Chain fields are never
counted, copied, marked, mirrored, or validated as language data.

We have these operations/helpers:
Path resolution uses `walkObservationPath(chain, path, onResolved)`. The final path segment is the optional target (or an empty path targets the root), while every preceding segment is required. Existing Errors are propagated; trying to dereference a missing/null/undefined or primitive required value produces a language Error. The COW mutation walker receives the same complete target path but remains separate because it installs copied branches while traversing.
import(obj, errorContext) - recursively walks an outside value's language-visible graph, marks every reached tracked value imported and shared, registers every unique promise immediately, and returns without waiting. Promise results re-enter the same walk with the same per-import WeakSet, recursively registering deeper promises before later mirror consumers run. Import performs no validation, counter work, mirror minting, or property writeback; validation remains lazy at counting time and mirrors remain the sole writers of settled properties. A non-promise root is returned synchronously and a promise root returns the derived promise. The required non-null errorContext (line/file/etc. in Cascada; a string here) is stored in each reached import marker and names later validation failures; null or undefined is a fatal caller error. Non-extensible values use META in WeakMap mode or the frozen side table in inline mode. Imported Error values remain language errors and count for hasError/getErrors/normalize. External objects are not expected to be mutated by outside code.
assignPath(chain, segments, value) - for changing the whole object or any path value. It mutates the Chain state and returns nothing; values are read only through lookupPath, normalize, hasError, or getErrors. Any branch/property with the shared mark will be copied-on-write. Import already registers promise properties for recursive marking, while mirrors are still created only by ordinary assignment, walk discovery, or ref-indexing and remain responsible for property writeback. The final property is the optional mutation target and may be added or replaced; every preceding value must exist and be trackable. A missing/null/undefined or primitive intermediate is replaced by a language Error instead of being autovivified. Path segments are objects, properties are assigned. The language-visible object surface is own enumerable string keys. Lookup treats `__proto__` and own non-enumerable keys as missing; mutations through `__proto__` throw a fatal runtime error; owned non-enumerable properties throw, while shared/imported branches COW first and shadow non-enumerables as missing; imported own enumerable `__proto__` keys fail at counting time; runtime writes stay plain `node[key] = value`. An Error at the target key is replaceable; an Error used as an intermediate path node is propagated and not entered.
deletePath(chain, segments) - for deleting a path. It mutates the Chain state and returns nothing. The final property may be absent, in which case deletion is a no-op; every preceding value must exist and be trackable, with a broken prefix replaced by a language Error. An empty path replaces the whole root with null. Array element deletion deletes the own index property and preserves array length, so later reads see it as missing/undefined. An Error at the target key is deletable; an Error used as an intermediate path node is propagated and not entered.
lookupPath(chain, segments, sharedOwnership=true) - returns the value. The final target may be missing and then returns undefined; a missing/null/undefined or primitive intermediate returns a language Error because another property was accessed through it. By default a returned tracked value is marked as shared, because it is now shared between its owning variable/path and the caller; because of this even root access has to go through lookupPath. If sharedOwnership is false, the lookup does not create shared ownership, so the returned object/array is not marked. This is valid for pure read/inspection, or when ownership is ceded to the caller as the last use of the original owner.
normalize(chain, segments, sharedOwnership=true, plainCopy=false) - resolves the path to the target branch with the same path rules as lookupPath (a promise along the way is continued through its mirror; an existing or generated Error along the path yields that Error; a missing final target or other untracked terminal is returned as-is; empty segments target the whole variable), waits for the promises inside the reached branch to settle, and hands out the branch's current state with all promises resolved - not an end-of-all-activity snapshot of the future. Marking follows "mark exactly what escapes, plus mark to pin a wait": a settled branch returned as itself is marked like lookupPath (imported provenance always, otherwise only when sharedOwnership=true), and normalize always marks when it must wait; a settled-with-errors branch returns a single Error unmarked, and settled plainCopy=true returns the deep copy unmarked (the copy is independent and produced inside the synchronous prefix). The pin mark is what makes "current state" exact for a pending branch: later operations copy-on-write away from the marked branch, so it can only settle, while suspended remainders of earlier operations still land in it through in-place mirror advances. Set the fourth argument, plainCopy, to true to perform a complete copy with no marks left. If there is any Error in the reached branch - the return is a single Error (just a single error indicating possibly many - this is a sandbox test implementation). Returns the answer synchronously when the branch is already settled at issue time, otherwise a promise that resolves to a complete value with no promises/errors in it, or an Error object if any Error or rejecting promise was still left.
hasError(chain, segments) - whether the property is an error, or there was an error along the path. A missing final target is false; a missing/null/undefined or primitive intermediate becomes a language Error and is true, while an existing Error is propagated. At the reached branch, validation failure or errorCount > 0 is true immediately; promiseCount === 0 is false immediately; otherwise it follows only the pending promises (descending only nodes with promiseCount > 0) and probes again at each settlement - first error wins, false once everything settles. A pure query: it never marks anything. The wait set is captured at issue time and extended only through resolved values read at hasError's own FIFO slots, so the answer is exact for the state when hasError was issued; later-issued installs are not observed, and later overwrite/delete of a watched pending key does not change the answer, though it may disconnect the eventual resolved value from the live tree.
getErrors(chain, segments) - returns the distinct Error identities reachable in the issue-time branch. It resolves the full path observationally: an existing Error along the path is propagated, while a broken intermediate produces a path-access Error. Any successfully reached terminal with no reachable Errors returns an empty array; this includes missing, null/undefined, primitive, clean extensible, and valid frozen terminals. The reached value is ref-indexed once; validation failure is returned as the sole Error. It follows the captured promise frontier recursively and waits for all of it, because the complete result cannot be known early. Counter-guided walking skips clean regions, one call-local Set deduplicates Error identities, and no shared mark or normalize settlement pin is created. A settled result is returned synchronously; otherwise a promise resolves to the final array after every captured and recursively exposed promise settles.

All of these run immediately - hasError, getErrors, and normalize too: their synchronous prefix ref-indexes the branch and registers at their program position. assignPath and deletePath return nothing. Observing operations return a value or a promise - a synchronous answer whenever it is decided before any suspension (e.g. error queries on a settled branch, normalize on an already-settled branch), a promise the program may await otherwise; we do not await one operation before issuing the next. One thing to keep in mind - the first to await will be the first to continue execution, so operations will run in order despite awaiting the same promises, there will be no race conditions. All operations (the actual calls: assignPath, lookupPath, normalize/etc…) to the same root shall happen sequentially - this is enforced by the implementer. Sequentially means issued and committed in program order, not awaited to completion: each operation's synchronous prefix runs to the end - committing its synchronous changes and registering its continuations at its own program position - before the next operation is issued. Suspended remainders complete later, in that same order per promise (FIFO). No operation is ever awaited before the next one is issued - that would serialize the program and make the FIFO machinery pointless.

Sharing is evaluated only from the point of the target language. Promises and resolved values are interchangeable (promises are implicitly resolved), none of the markings are visible, etc… The shared mark is a field of the runtime's per-node metadata record, a single non-enumerable Symbol property ignored by the language (see Subtree counters - Metadata record). Non-extensible external objects, such as frozen/sealed objects, are treated as already safe for COW without adding any metadata.

A promise exposed through shared-ownership lookupPath/import carries the same SHARED obligation as a directly returned object/array. If it resolves to an object/array, the resolved value is marked SHARED before consumers can observe it.

Shallow Copy - a COW copy duplicates the language-visible enumerable string keys of the object/array into a fresh object/array, preserving array length. As the object that requires such copying is shared, the copy is the new owned/unmarked node. The source object keeps any shared mark it already has; marks are never removed. Reused non-path child objects/arrays copied from a shared branch are also marked, so shared references stay protected. The current path child is tracked by the walk's inherited shared state and becomes owned/unmarked after it is replaced or copied for the current mutation, unless the assigned/reused object already carried its own shared mark. Another, simpler approach is to deep-freeze the whole copied branch. And a third approach is to add both SHARED and NOT_SHARED markings, and using not-shared for the new along-the-path properties. The copy never inherits the source's metadata record - it gets a freshly constructed one (owned/unmarked, fresh mirror map, counter snapshot; see Subtree counters - commit points, shallowCopy). Every copied key that holds a promise is forked eagerly at copy time (see Value resolve - the Fork birth point); non-path promise captures are marked shared, while the current path promise is protected by inherited shared state when the walk advances through it. A copy of an imported node additionally propagates the import marker (with its errorContext) to every reused tracked child, the path-key child included - provenance is about origin, not aliasing, so it survives the copy even where the shared-mark optimization skips the path key; forked mirrors inherit the source mirror's import flavor. A copy of a frozen source that holds a promise key seeds the fork mirror from the raw promise instead of a source mirror: no mirror can attach to a non-extensible node and no writeback can ever advance it, so the raw settled value is the only version that will ever exist. If that frozen source is imported, the raw-seeded mirror takes the walk's current import context, so its settled value is still marked imported+shared before consumers observe it. If the source has an own enumerable `__proto__` data key, the copy pre-creates the same own data key before the normal assignment loop, so no prototype mutation occurs; promise-valued `__proto__` keys are preserved raw and do not receive mirrors because later writeback would be a forbidden mutation. The scan is O(width of the node), the same order of cost as the copy itself.

Shared Marking (requiring copy-on-write) - Import and shared-ownership lookupPath create shared values. Import materializes an imported/shared mark on every tracked node it reaches, including nodes recursively exposed by promises; this makes external DAG identities directly recognizable and lets error queries visit them once. A shared-ownership lookup marks the returned branch root. Its descendants are logically inside a shared branch but need not carry direct marks, so mutating walks still track inherited shared state. Lookup extraction from imported data preserves the import marker even with sharedOwnership=false, COW stamps reused imported children, and import-flavored mirrors preserve the context before writeback. Non-extensible objects are implicitly shared and keep imported attribution in META under WeakMap storage or the frozen side table under inline Symbol storage. All operations that pass, return, or store a whole existing object must use sharedOwnership=true unless they can prove ownership transfer or pure inspection.

Value resolve - each node can have a hidden promise mirror map, held in the per-node metadata record (see Subtree counters - Metadata record), created for every key that was assigned a promise. (Non-extensible nodes never carry mirrors: a promise key on a frozen holder is read mirror-free through the raw settled value - no advance is ever possible there - and is rejected by counting-time validation, so the inline record always suffices.) Each promise mirror stores {promise, currentValue}: the exact promise assigned to that key, and the newest version of its resolved value. `currentValue` is changed by each operation that mutates it, in operation order, so each op's change is applied on top of the previous `currentValue` rather than on the original resolved value. When an advance commits through a ref-indexed parent, it stores the value returned by the write helper: validation may replace the candidate with an Error, and later FIFO continuations must observe that committed Error rather than the rejected candidate. A normal owned promise writeback does not mark the resolved object/array shared; the resolved value is still owned by the single Cascada location that received the promise. Import and shared-ownership lookupPath mark resolved objects/arrays shared because they cross an ownership boundary. Forked non-path promise mirrors also mark the captured value because a copied world and the source world initially reuse that same `currentValue`; a forked current-path promise instead COWs from the walk's inherited shared state if the walk advances through it. The resolved value is written back into its property only if its promise mirror is still the one the map holds for that key. If a later op has since reassigned the key (even to the same promise) - a new promise mirror is created. Even when the mirror was revoked, a ref-indexed former parent requires its private logical child to be validated and ref-indexed before `currentValue` becomes observable; this changes no live property, parent edge, or count. The private value remains alive for reads that captured the mirror. Every synchronous overwrite/delete through the runtime must clear the promise mirror map for that key, even when assigning the same value. All required accessible promises must start resolving immediately: we use .then and continue walking rather than await + enter, leaving other immediately accessible promises to resolve in parallel (and yet in operation order as guaranteed by JS).

A promise mirror is born at exactly three points, and where its starting `currentValue` comes from differs:

- Assignment: a promise is assigned to a property (assignPath value, literal initialization). A fresh promise mirror is always created, even if the same promise instance was already at this key - two assignments are divergent worlds and must not share `currentValue`. The writeback continuation is registered immediately at assignment, before any operation can register on that promise for this key, so the writeback is always the first responder and `currentValue` starts as the raw resolved value.

- Discovery: a walk or operation reaches a promise key that has no mirror (imported data, raw literals - any orphan). The mirror is created and its writeback registered at the discovering operation's program position; `currentValue` starts as the raw resolved value. A mirror discovered while a lookup or ref-indexing walk carries import context records that context, and its writeback marks the settled value imported+shared before any consumer can observe it - settled external data is a fresh boundary crossing.

- Fork: a copy-on-write shallow copy duplicates a key that holds a promise. The copy's promise mirror must be created eagerly, at copy time - never minted lazily by a later walk. A lazily minted mirror would seed `currentValue` from the raw resolved value, silently losing every mutation made by operations issued before the copy. At copy time, for each copied key that holds a promise: ensure the source node has a mirror (Discovery, if it is an orphan), then create the copy's mirror and register its initializer immediately, at the copier's program position. This initializer never reads the raw resolved value - it reads the source mirror's `currentValue` at its own FIFO slot and uses it as the copy's starting `currentValue`, mirroring it into the copy's property only if the copy's key still holds the promise unchanged. Non-path captures are marked shared because they are now reused by both worlds; the current path capture is left unmarked and relies on the walk's inherited shared state if the mutation continues through it. Because the initializer is registered at the copier's program position, it observes exactly the advances of operations issued before the copy and none issued after - the two worlds diverge at precisely the copier's place in program order, uniformly for pending and settled-but-unreplaced promises. After the fork the mirrors are independent: operations traversing the copy advance the copy's mirror, operations traversing the old node advance the source mirror, and either a shared mark on the captured value or inherited shared state forces the first advance that would mutate a reused value to copy. Value advances themselves go through the same shallow copy, so a resolved value that contains nested promise keys is forked by this same rule automatically.

assignPath - Receives the complete target path: the final segment is the optional target property, every preceding segment is required, and an empty path targets the root. If it reaches a promise before the target it resolves it and continues through the mirror's `currentValue` (see Value resolve), COWing that value and advancing the mirror. If a required intermediate is missing/null/undefined or primitive, replace that intermediate with a path-access Error and stop; there is no autovivification. Continue through the path (resolving any promises along the way), assigning or creating the target property when the parent is reached. Because it is the first to await the promise, it is the first to resume and modify. When it reaches an object marked shared along the path it copies it (the COW approach above); non-enumerable properties are not copied, so writes shadow them as missing. Mutating through `__proto__` or through an owned own non-enumerable property throws a fatal runtime error. assignPath into an intermediate Error propagates that Error unchanged, while assignment at an Error target replaces it. The Chain's `_state.value` slot is the authoritative mutable root location: copied/replaced roots are installed there synchronously, while mutations through a pending root leave the slot at the promise until the live root mirror writes back the settled/current value. assignPath returns nothing; callers keep using the Chain as the root location.

lookupPath - returns the value it reaches. It reads only own enumerable properties; `__proto__` and own non-enumerable properties are treated as missing. If the reached value is a promise, or if a promise lies along the path, it returns a promise that resolves to the value once reachable - continuing through `currentValue` if the path was mutated during resolution (see Value resolve), never the raw resolved value. The returned promise is not awaited by lookupPath itself; values resolve only at the point of consumption. If sharedOwnership is true, marks the return shared if it is an array or object; if the return is a promise, it ensures the resolved value is marked shared. sharedOwnership=false lookups are for inspection or final ownership transfer and do not share ownership. Path and Error handling otherwise follows assignPath. Any Error or rejecting promise along the path results in an Error return.

deletePath - resolves every required parent-path segment (a segment may be a promise, continued through `currentValue` per Value resolve, not awaited), then deletes the optional target property, clearing its promise mirror so no later writeback re-mirrors it. A missing target is a no-op; a missing/null/undefined or primitive intermediate is replaced by a path-access Error. COWs shared branches along the way (the COW approach above), shadowing non-enumerables as missing after the copy. Array targets are normal property deletes: the index property is removed and length is preserved. Mutating through `__proto__` or through an owned own non-enumerable property throws a fatal runtime error. Any existing intermediate Error is propagated unchanged, while an Error target can be deleted. deletePath returns nothing; the Chain remains the authoritative root location.

Optimizations:

Subtree counters (implementation - supersedes the CLEAN flag, and settles the RefCounting question: we count promise/error content, not references; ownership stays mark-based). Every ref-indexed node carries exact counts of the pending promises and Error values reachable in its subtree (ref-indexing is lazy and branch-level - see below). hasError becomes O(1) at a settled node, and normalize's completion becomes an event - the watched promiseCount reaching 0 - instead of repeated rescanning. CLEAN survives only as a derived property: clean ⇔ promiseCount + errorCount === 0. Counters answer "what is pending/broken below me"; the SHARED mark answers "who else can see me" - the two never substitute for each other.

Metadata record - all runtime metadata lives in one per-node record, stored either under a single non-enumerable Symbol or in a WeakMap according to `STORE_META_IN_WEAKMAP`, selected by `CASCADA_META_STORAGE` (`weakmap` selects WeakMap; inline Symbol is the default). Inline Symbol lookup is own-property-only: a runtime object used as another object's prototype must never lend its META record to the derived object. The record is ignored by the language and never copied by the shallow copy. The SHARED flag and the promise mirror map share this record with the counters rather than owning separate Symbol properties. Under lazy branch-level ref-indexing, record existence does not imply counted-ness: mirrors, shared marks, and import markers can create a record before counters are live. The complete test matrix runs in both storage modes.
- shared - the SHARED mark; set once, never cleared; false at birth is the old "no mark". hasSharedMark(v) = meta(v)?.shared === true || !Object.isExtensible(v).
- mirrors - the promise mirror map (see Value resolve), lazy.
- promiseCount, errorCount - the counters, valid only when the node is ref-indexed. promiseCount: promise-valued own keys contribute 1 each; each tracked child edge contributes the child's promiseCount times the edge multiplicity. errorCount: the same for Error values.
- parents - exact reverse edges and the ref-indexed marker. `parents === undefined` means counters are not live; an empty `Map` means ref-indexed root / no ref-indexed parents; a populated `Map` is `Map<parentNode, edgeCount>` (a parent referencing the same child under two keys has multiplicity 2). Every commit that creates or removes a parent/child edge updates it; when an edge count reaches 0, delete that parent entry, never the `parents` map itself. References are strong in the sandbox (a world held by a suspended reader must keep receiving deltas); WeakRef edges pruned on dead deref are required in the real implementation, sound because liveness then coincides with observability - with strong edges every COW'd-away world stays referenced by its reused children and keeps receiving deltas, so per-delta cost grows with the COW history of a hot key: a time tax, not just memory.
- settlementPromise - optional shared promise generation added only by normalize while callers are waiting for promiseCount to verify at 0; hasError uses no settlement wait - its pending handling is the per-call Promise.all tree returned by probeIndexedBranchForErrors, not META fields.
- settlementResolve - resolver for settlementPromise.
- settlementVerifyScheduled - boolean latch that coalesces settlement verification callbacks while settlementPromise exists.
- importContext - the import marker and error attribution in one field: undefined means not imported; otherwise the required error/attribution context of the import that brought the data in. The recursive import walk sets it on every reached tracked value; extraction, COW reuse, and settle boundaries preserve it, and first import wins. Setting it also sets the shared mark. With WeakMap metadata, non-extensible imported nodes use this field; inline Symbol storage uses the scoped frozen side table, with inherited context as a fallback for later runtime paths.

Frozen rule: non-extensible (frozen/sealed) objects are forbidden to contain promises or Error values anywhere beneath them in the language-visible enumerable string-key graph - enforced at counting time only (validateCountable), because that is the only place it matters: a non-extensible node permanently counts as [0,0], so anything pending or broken beneath it would make the counts lie. A violation surfaces with import-context attribution - an Error committed at a ref-indexed write key, or returned from normalize / true from hasError. Outside counting, frozen data is handled gracefully: reads through a frozen holder's promise key resolve mirror-free (no advance is possible, the raw settled value is the only version), and a COW of a frozen source seeds its fork mirror from the raw promise, producing a normal mutable copy. A non-extensible node cannot be mutated in place and never carries counters, parents, or promise mirrors. It may carry only import/shared metadata when storage is WeakMap-backed; inline Symbol storage uses the frozen import-context side table instead.

getRefCounts(value): pending or settled-but-unreplaced promise [1,0]; Error [0,1]; non-extensible node [0,0] - trustworthy only after validateCountable has accepted the frozen subtree, which every path into a counted region guarantees (branch validation descends frozen children, ref-indexed writes validate entering values, buildRefIndex validates frozen roots); tracked extensible node its stored totals, with a missing `parents` field treated as a fatal downward-closure violation rather than repaired by implicit ref-indexing; primitive [0,0]. Counts are purely structural - a function of what is physically at the keys - and change only at commit points, in the same FIFO slots that mutate the tree, so they inherit the ordering guarantees.

applyCountDelta(node, dPromise, dError): add to the node's counters, then recurse into every parent multiplied by edge count; zero deltas short-circuit. Propagation termination relies on acyclicity, which is thereby promoted from an optimization assumption to a hard invariant (see Cycles).

Lazy, branch-level ref-indexing - counter metadata exists only where normalize/hasError have been used; uncounted data pays zero counter bookkeeping (promise mirrors and import/shared marks are independent). The first normalize/hasError on a branch ref-indexes exactly that branch, at the caller's program position; ref-indexing stops at sub-branches that already carry counter metadata, registering the boundary edge and reusing their totals; from then on the ref-indexed region is maintained forever. Deltas stop naturally at a region's top (no parents above); ref-indexing an ancestor branch later connects the regions through the boundary edge. Two load-bearing rules: (1) ref-indexed regions are downward-closed - ref-indexing walks whole trusted subtrees and a write into a ref-indexed parent ref-indexes the entering value before commit. (2) The ref-indexed gate is evaluated at commit time, never captured at registration: a continuation registered while a node was not ref-indexed performs full bookkeeping if the node is ref-indexed when it commits; overlapping ref-indexing attempts over the same pending promise are safe by idempotence (`parents` is created only once, and the mirror map reuses the same mirror for the same promise - no duplicate writebacks or marks). Copies inherit the source's ref-indexing. Validation is lazy and lives at counting time: import recursively marks and registers promises but validates nothing. The first counter check pays its own O(branch) validation/ref-index scan; repeated checks are O(1). A ref-indexed write commit additionally pays a validation descent over the entering value.

Ref-indexing / counting-time validation - ref-indexing initializes counters bottom-up, registers parent edges, and mints a promise mirror (Discovery, import-flavored inside imported regions) for every promise-valued key in a ref-indexed region. Eager minting is required because nothing rescans ref-indexed regions: an orphan promise with no registered writeback would hold promiseCount up forever. Validation happens exactly where counting needs its guarantees, through one pure walker, validateCountable(value, writeTarget): two-color cycle marking (reaching a visiting node is a cycle; a visited node is a DAG share), the frozen rule, and - when a writeTarget is given - the back-edge check (node === target; sufficient, because a write-created cycle must reach the written parent). It runs as the validate pass of buildRefIndex (no target; already-ref-indexed subtrees may be skipped only in ordinary mutable context, because a non-extensible ancestor's stricter no-promise/no-Error rule must revalidate them) and of every ref-indexed write commit (target = written parent; no early exit, the target may hide behind indexed shares). The cycle check covers trusted nodes too: a cycle can thread through owned data via an in-place writeback in an uncounted region, so only the whole-branch walk is sound - and the per-node facts it needs are O(1). Validate-then-commit stays two-pass: the validate pass is pure, the commit pass cannot fail, so a rejection leaves no partial counters, edges, or mirrors. Failures carry the import marker's errorContext. A back-edge writeback under a non-ref-indexed target commits uncaught: the cycle floats harmlessly in the uncounted region (every walk is path-bounded) and is rejected, with attribution, when counting first reaches it. Internal operations cannot create cycles when the compiler obeys the ownership contract: RHS/escaping object values go through shared-ownership lookupPath, so `a.property = a` stores a COW copy of `a` as it existed before `property` was added.

Commit points - every runtime property write or delete is performed by `src/index.js`. Before the write/delete, `src/index.js` calls the refcount hooks `refSetProperty(parent, key, value)` and `refDeleteProperty(parent, key)`, which own exactly the counter bookkeeping: ref-index an entering value when needed, delta at the written node (new property → +getRefCounts(new); replaced property → −getRefCounts(old) +getRefCounts(new); delete → −getRefCounts(old), the cleared key's contribution simply removed), parent-edge swap, propagation. The refcount hooks never perform the language write/delete; `refSetProperty` returns the value that should be written, because validation can turn an entering value into an Error. `refIndexChildValue(parent, value)` exposes the same validation/ref-indexing gate without a property or counter update for a revoked mirror's private logical child. Mirror lifecycle (fresh mirror on promise assign, guarded live writeback, clear on overwrite/delete) remains in the mirror layer. The caller sites:
assignPath target - overwrite; a newly assigned pending promise contributes [1,0], its mirror registered at assignment as before.
deletePath target - subtract the removed value's counts, remove the edge.
Promise settlement/writeback - when the mirror still owns the key, prepare the child through the ref-indexed write gate, then replace [1,0] with getRefCounts(resolved value). When the mirror was revoked but its former parent is ref-indexed, validate the resolved value against that parent and ref-index the private child without touching the live property, edges, or counts. In both cases `mirror.currentValue` is prepared before later FIFO consumers run, so observers never perform fallback ref-indexing. A suspended commit into a disconnected branch runs the same code and propagates only through that world's ancestors - exact per world.
Walk installs - the synchronous next !== child replacement and the suspended parent[key] = next, same rule.
shallowCopy - the copy copies language-visible keys and forks promise mirrors as today, then calls `copyCounters(source, copy)`. That hook is a no-op unless the source is ref-indexed; ref-indexed copies snapshot counter totals, receive `parents = new Map()` (ref-indexed copy, no ref-indexed parents yet), and register themselves as a parent on every reused tracked child.

normalize(chain, segments, sharedOwnership, plainCopy) - the path resolves through the observational, context-threading `walkObservationPath` (shared with hasError - callers decide marking after resolution, while the resolver threads importContext for mirror flavoring and failure attribution; ref-indexing at the operation boundary receives that inherited context). An Error or untracked target is returned as-is, uncounted and unmarked. Otherwise the reached branch is ref-indexed and, following "mark exactly what escapes, plus mark to pin a wait", marked at normalize's program position when it will be returned as itself with shared ownership or waited on (markImported with context inside an imported region; otherwise markShared only when sharedOwnership=true, except pending normalize always marks to pin) - the pin mark is what makes completion count-driven and exact; a branch that later collapses to Error keeps its pin mark, since the outcome is unknowable at call time. normalize and hasError are ordered like every other operation: issued at their program position, their synchronous prefix ref-indexes the branch (activating the counter metadata) and registers on its pending promises there, and their remainders resume through the uniform wrapper in FIFO order, interleaving correctly with other operations' continuations on the same promises. The runtime never schedules outside the wrapper. Completion is branch settlement (promiseCount reaching 0) - exact because of the call-time mark: later-issued operations COW away from the marked branch and cannot add promises to it, while suspended remainders of earlier operations still land in it through in-place mirror advances; the wait-set is precisely the promises present at the call plus, recursively, promises arriving inside their resolved values. No early exit on errors: an earlier-issued remainder can still replace an error before settlement, so collapse-to-Error is decided only at zero. Both ops build and thereafter maintain the counter metadata - the first call on a branch pays the ref-indexing scan, later calls are O(1) plus path resolution. hasError never sets the SHARED mark - a pure query must not change ownership state; normalize marks exactly what escapes (plus the pin for a pending wait), and the call-time pin is what makes the snapshot semantics true. Like every operation, normalize returns a value or a promise: if promiseCount is already 0 at issue time the answer is returned synchronously (errorCount > 0 → a single Error, unmarked; plainCopy=true → the deep copy, branch unmarked; otherwise the branch, marked only when imported/sharedOwnership says it escapes) and no settlement wait is ever created. A shared `meta.settlementPromise` generation exists only while promiseCount > 0 and normalize callers are waiting; `meta.settlementResolve` holds its resolver; `meta.settlementVerifyScheduled` coalesces the queued check; the only wake source is the zero-crossing inside applyCountDelta. Liveness holds because every decrement of the pinned branch's counters happens inside a continuation of one of its counted promises: later-issued operations COW away and cannot touch it, while earlier-issued remainders ride counted promises - their writebacks and their COW installs at mirrored keys (which may remove counted promises without settling them) all run as continuations of promises the branch counts. Verification is registered through the uniform wrapper on `Promise.resolve()`; it never fires synchronously and never through a bare microtask. The ordering is safe without knowing which promise is settling: at the moment of the crossing, every consumer already registered on the settling promise is already enqueued, so the verification runs after all of them - and the deeper invariant is that any earlier-issued remainder still able to land in the marked branch must ride a promise physically counted [1,0] inside it, so a true zero means nothing earlier can still arrive. On run, re-check and fire only if still 0; otherwise the next zero-crossing re-schedules. This is the counter form of "repeat scanning until settled": identical semantics, O(1) per iteration. On fire: errorCount > 0 → a single Error (sandbox collapse); plainCopy=true → deep copy through an old→new identity map consulted before recursing - one map serving two stated rules: visited-gate (termination) and identity-reuse (lookup-shared diamonds are DAGs and must not duplicate; COW deliberately does not preserve internal aliasing, normalize(..., plainCopy) deliberately does); otherwise return the already-pinned branch. Whenever the branch is marked due to import, a branch reached inside an imported region is marked imported with its context, not merely shared - provenance is owed on escape; failure attribution never depends on marking, because ref-indexing receives the walk's inherited context directly. A non-extensible root never receives metadata or creates settlement waits: validate it with validateCountable (pure, repeatable - frozen data is static), then treat it as already settled [0,0] - a violation returns the Error with import attribution, plainCopy copies it plainly, otherwise it is returned as-is (already safe for COW).
hasError(chain, segments) - observational `walkObservationPath(chain, segments, onResolved)` treats the final path segment as the optional target. Missing targets arrive as `undefined` and return false; missing/null/undefined or primitive required intermediates arrive as a generated path-access Error and return true; existing Errors are propagated. The walk is pure inspection: only a boolean escapes, so no extraction marking, but it still flavors discovered mirrors. Like every operation hasError returns a value or a promise: a bare boolean whenever the answer is decided before suspension. At the reached branch it uses no shared settlement wait and never marks; the captured wait tree and slot-exact mirror reads give issue-time answers without a pin. `hasErrorAtPathValue` builds the generic ref index only at the resolved path-value boundary, then `hasErrorInIndexedBranch` answers synchronous results directly from `errorCount` and `promiseCount`. Only a pending branch allocates the final boolean race and calls `collectErrorSearchWaits`; after `buildRefIndex` has registered writebacks, each pending key contributes `onValueResolve(childPromise, () => probeResolvedPromiseForErrors(mirror.currentValue, resolveError))`. Behind those promise boundaries, `probeIndexedBranchForErrors` checks the already-indexed resolved branch and returns any nested wait tree. hasError races the clean `Promise.all` tree against `errorPromise`. Each settlement continuation consumes the already-prepared mirror value directly: Error calls the resolver, primitive and valid non-extensible values are clean, and tracked extensible values go straight to `probeIndexedBranchForErrors`. The whole promise tree resolving means the clean side is false. A validation failure signals true (invalid countable data is an error condition for a boolean query). Because hasError does not pin, later-issued installs are not observed; later overwrite/delete of a watched pending key can detach the resolved value from the live tree, but does not change the answer for the issue-time branch. A reached frozen branch is validated instead of probed, with validation failure returning true. Imported Error values count as language errors.

Error-query deduplication is operation-local and precise. One `visited` WeakSet spans recursive promise slots, and only nodes with their own shared status are checked and added. The weak set does not retain settled branches while another captured promise remains pending. Every valid reused identity is marked by the ownership, import, or COW boundary that created the reuse; an unmarked node is owned and cannot recur in valid data. A recurrence through an ancestor would be a cycle, which the initial ref-index build or resolved-promise writeback rejects before the query probes that branch.

getErrors(chain, segments) - resolves the full path observationally, builds the generic ref index only at the reached path value, and returns each reachable Error identity once. Any successfully reached terminal with no reachable Errors returns an empty array, including missing, null/undefined, primitive, clean extensible, and valid frozen terminals. An existing Error along the path, a generated path-access Error from a broken prefix, or a counting-time validation failure becomes the sole result. `collectErrorSearchWaits` is shared with hasError: both follow counter-pruned promise regions, while getErrors also follows counted Error regions and adds Error leaves to one call-local Set. Every captured promise continuation consumes the already-prepared `mirror.currentValue` and returns any nested wait promise, so getErrors waits for the complete issue-time promise tree before materializing the Set. It does not mark, pin, race, or call `buildRefIndex` behind promise boundaries; live and revoked mirror writeback has already indexed those resolved branches.

Correctness:

Cycles: counted regions can never contain cycles.
a.property.subproperty = a
The moment we read `a` through lookupPath it becomes shared. The moment we modify subproperty - `a` becomes copied (simpleAssignPath), thus it is a new object, no cycle. The recursive import walk terminates on cyclic external data but does not reject it; counting-time validation rejects it when normalize/hasError or a ref-indexed write first needs the branch, with the import marker's errorContext naming the import site. One back door: external code can resolve a promise with a language object it received earlier - for a ref-indexed target the write commit's back-edge check rejects it (an Error is committed instead of the back-edge); for an uncounted target the cycle floats harmlessly (every walk is path-bounded - path operations still terminate) until counting validation rejects the region. With subtree counters, acyclicity of counted regions is load-bearing for delta-propagation termination, not merely for walk termination - which is exactly why validation is anchored to counting.

Object reuse: Any object that is reused - the runtime-visible ways this can happen are import and shared-ownership lookupPath, which mark the object as SHARED. Import marks the complete recursively exposed external graph, so internal and async DAG aliases carry direct marks. An assignment and initializer can target only one variable, so ownership is clear: `var x = {a:1, b:2}` is owned by x and needs no mark. An async assignment to one variable is still owned by that variable after resolution. `var y = x` obtains x through shared-ownership lookup, marking the reused value. A pure read or final ownership transfer uses sharedOwnership=false.

Race conditions: with assignPath and deletePath, it may look as if we are opening the possibility for race conditions. While one operation awaits a promise that resolves some data, a subsequent operation may modify the data - or commit synchronously and overtake the suspended one entirely. Why neither can corrupt anything or diverge from sequential semantics:

Program order = FIFO order. Operations are issued in program order and register their continuations at their own program position; JS runs continuations on one promise in registration order. So all consumers of one promise mirror resume in program order, each advancing the mirror's `currentValue` on top of the previous operation's version (V → V′ → V″). Immediate resolution of all accessible promises is required for this to hold (use .then and keep walking, never await + enter).

Owned promise writeback does not mark by itself. Escape marking (shared-ownership lookupPath/import) and fork marking are what force advances to copy instead of mutating a value that another consumer or forked world still references.

Suspended reads are immune to overtaking writes: a suspended lookupPath continues through the mirror's private `currentValue`, never the live property. A later operation that synchronously reassigns or deletes the key (clearPromiseMirror + assignment) is invisible to it - the read resolves to the mirror value as of its own program position, exactly what sequential execution would have returned.

Suspended writes are fenced by the promise mirror map: a resumed write mirrors into the live tree only if its mirror still owns the key (map[key] === mirror - revoked by a later assignment or clearPromiseMirror). A revoked write completes on the disconnected mirror value - a wasted copy, never corruption - matching sequential semantics, where the later operation's assignment or delete overwrote that write anyway. In particular, a delete followed by a resumed write cannot recreate the deleted path.

Reassignment or delete at an ancestor orphans the whole subtree instead: local mirrors inside it may still succeed (the guard is deliberately local), but every effect is confined to the disconnected branch - a waste, not a validity issue - and the live tree matches sequential semantics, where the ancestor operation discarded that branch anyway.

COW forks of keys holding promises cannot reorder anything either: the copy's mirror is seeded from the source mirror's `currentValue` at the copier's own FIFO slot and the captured value is marked shared, so the copied world contains exactly the writes of operations issued before the copy, and operations issued after advance the two mirrors independently through copy-on-write (see Value resolve - Fork).
